# Монорепо: узкие PR и меньше шума в `git status`

## Ветки и объём диффа

- Правки **`_ai-share/synth-1-full`** оформляйте отдельной веткой и отдельным PR, без случайных изменений в корне **`Projects/`** (Python, корневые workflow, чужие подмодули), чтобы ревью оставалось по теме.
- Если в корне появились только локальные/генерируемые файлы (IDE, артефакты), не коммитьте их в ту же ветку с продуктовыми правками.

## Локально скрыть не относящиеся к задаче файлы

- Локальный **`.envrc`** для **direnv** не коммитится (см. **`.gitignore`**); в репозитории есть шаблон **`.envrc.example`**.
- **`.git/info/exclude`** — как `.gitignore`, но только на вашей машине (не в репозитории). Удобно для персональных путей, которые нельзя добавить в общий ignore.
- Каталог **`.cursor/`** или локальные overrides — держите вне коммитов, если это не договорённая часть команды.

Коммитить в общий репозиторий имеет смысл только осознанные правки в **`_ai-share/synth-1-full/**`** и сопутствующие изменения в корневом CI (**`.github/workflows/synth-1-full-ci.yml`**, шаблон PR), если они относятся к тому же изменению.
