# Ритейл-кабинет: сквозной контур, MVP и роли

Документ фиксирует договорённости по продукту и синхронизирован с навигацией (`shop-navigation-normalized.ts`) и режимом `NEXT_PUBLIC_SHOP_NAV_MVP`.

## 1. Сценарий сквозной ценности (деньги)

**Цепочка:** черновик заказа бренду → подтверждение → оплата → поставка (сроки, трек) → оприходование на РЦ/магазин → маржа.

В UI это отражено как **один процесс «Поставки»** в блоке исполнения опта: сроки и каналы, автопополнение, отслеживание, остатки по сети — подразделы одного пункта, а не четыре независимые «философии».

Розница и опт **не смыкаются юридически**; смыкаются **данными** (остаток, маржа, клиент там, где это допустимо по роли).

## 3. Критерии готовности (не «экран сделан»)

- Заказ опта меняет статус в учёте или есть явная очередь синхронизации с ошибками.
- После отгрузки/оприходования остаток сходится с правилом (или видно расхождение).
- У денежных операций виден статус обмена с 1С/ERP (время последнего успеха — целевое поведение UI).

## 5. Режим навигации MVP

`NEXT_PUBLIC_SHOP_NAV_MVP=1` — в сайдбаре скрыты пункты с `navTier: 'phase2'` (см. код). По умолчанию режим полный (`full`).

## 6. Что убрать из доверия к кабинету

Пункты без закрытого цикла (нет статуса, роли, интеграции) помечать как Phase 2 или не показывать до готовности бэкенда.

## 7. Матрица ролей → сайдбар (реализовано в коде)

| Роль (PlatformRole) | Обзор | Розница | Склад сети | Закупка / исполнение / сервис B2B | Аналитика | Сеть и доступы |
|---------------------|-------|---------|------------|-----------------------------------|-----------|----------------|
| **retailer** | ✅ | ✅ | ✅ | ✅ (`b2b_orders`) | ✅ | ✅ (`settings`+`team`) |
| **buyer** | ✅ | ⛔ (`SHOP_NAV_GROUP_ROLES`) | ⛔ | ✅ | ✅ | по `can(settings)` |
| **finance_manager** | ✅ | ⛔ | ⛔ | ✅ (оплата/заказы) | ✅ | ✅ |
| **distributor** | ✅ | ✅ | ✅ | ✅ | ✅ | ✅ |
| **sales_rep** / **merchandiser** | ✅ | ✅ | ✅ | ✅ | ✅ | по `can(settings)` |
| **admin** | ✅ всё | ✅ | ✅ | ✅ | ✅ | ✅ |

Уточнения: `SHOP_NAV_GROUP_ROLES` в `profile-page-features.ts` режет **розницу** и **склад** для buyer и finance_manager; остальные группы — по `SHOP_NAV_GROUP_RESOURCE` и матрице `rbac.ts`.

## 8. Локально: урезанное меню (MVP)

В `.env.local`: `NEXT_PUBLIC_SHOP_NAV_MVP=1` — скрыты пункты с `navTier: 'phase2'` в `shop-navigation-normalized.ts`. Перезапуск dev-сервера обязателен.

## 9. UI: учёт и ID заказа опта

- Блок **«Учёт: последний успешный обмен…»** (`ErpAccountingSyncBanner`) на экранах списка/карточки заказа, оплаты, нового заказа, landed cost; данные — `getErpAccountingSyncStatus()` (пока демо, заменить API).
- **ID заказа опта** — компонент `B2bOptOrderIdCopy` на карточке заказа; в списке заказов колонка и ссылка на детализацию.

## 10. API статуса учёта

- **GET** `/api/shop/erp-sync-status` — JSON с `lastSuccessAt`, `systemLabel`, `pendingInQueue`, опционально `lastError`, `health`, `connectorId`, `syncDurationMs`, `lastSyncJobCompletedAt`, `domainOutboxPending`, `domainOutboxFailed`.
- Сервер: async `getErpAccountingSyncStatus()` в `erp-accounting-sync-service.ts` (env + коннектор platform + доменный outbox).
- **Env (сервер):** `ERP_ACCOUNTING_SYSTEM_LABEL`, `ERP_SYNC_LAST_JOB_COMPLETED_AT`, `ERP_SYNC_LAST_SUCCESS_MINUTES_AGO`, `ERP_SYNC_PENDING_QUEUE`, `ERP_SYNC_LAST_ERROR`.
- Клиент: `ErpAccountingSyncBanner` запрашивает API; при ошибке — локальный fallback и жёлтая рамка.
- На странице `/shop/b2b/shopify-sync` ссылка «Синхронизация» скрыта (`hideSettingsLink`).

## 11. Слияние с интеграциями и расширенные поля

- `lastSuccessAt` берётся как **более позднее** из env-демо и `lastSync` коннектора `platform` в `getB2BIntegrationStatus()`.
- Поля **`health`**, **`connectorId`**, **`syncDurationMs`** — в ответе API и в баннере (вторая строка мелким текстом).
- Доп. env: `ERP_SYNC_CONNECTOR_ID`, `ERP_SYNC_DURATION_MS`.
- Баннер добавлен на **checkout**, **tracking**, **margin-report**, **margin-calculator** (кроме уже подключённых экранов).

## 12. Единый срез аналитики (розница / опт / маржа)

- Базовый компонент `ShopAnalyticsSegmentNav` (три ссылки: `/shop/analytics`, `/shop/b2b/analytics`, `/shop/b2b/margin-analysis`); обёртка **`ShopAnalyticsSegmentErpStrip`** добавляет баннер учёта (кроме сводки `/shop`, где только срезы).
- Активная вкладка: розница — `/shop/analytics*` (не B2B); опт — `/shop/b2b/analytics`, `/shop/b2b/order-analytics`, `/shop/b2b/reports`; маржа — `/shop/b2b/margin-*`.
- Встроен strip в: сводку `/shop`, розницу, footfall, B2B-аналитику, order-analytics, reports, margin-report, margin-calculator, margin-analysis.

## 13. Реализация (обновление)

- **`ShopAnalyticsSegmentErpStrip`** — один импорт: переключатель срезов + `ErpAccountingSyncBanner`; на `/shop` только срезы (`showErpBanner={false}`).
- **`getErpAccountingSyncStatus()`** — async, `server-only`; к env и коннектору `platform` добавлены **`domainOutboxPending`**, **`domainOutboxFailed`** из `getDomainEventOutboxStats()`, опционально **`lastSyncJobCompletedAt`** из `ERP_SYNC_LAST_JOB_COMPLETED_AT`.
- **E2E:** `e2e/shop-erp-analytics-strip.spec.ts` — GET `/api/shop/erp-sync-status`, видимость `data-testid="shop-analytics-segment-nav"` на `/shop/analytics`, `/shop/b2b/order-analytics`, `/shop/b2b/reports`.

- **Хаб `/shop/b2b/margin-analysis`** — страница-вход в срез «Маржа» (ссылки на отчёт, калькулятор, landed cost, fulfillment); strip `ShopAnalyticsSegmentErpStrip`.
