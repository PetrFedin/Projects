# **App Name**: Syntha AI

## Core Features:

- User Authentication and Roles: Firebase Authentication integration supporting Email/Password, Apple, and Google sign-in methods. Implements Role-Based Access Control (RBAC) with editable roles: client, brand, shop, admin, and analyst. Roles and permissions are managed dynamically via the Admin Console, with mappings stored in Firestore (/users/{uid}.roles). Includes onboarding flow for each role and a built-in Role Switcher (admin-only).
- Product Catalog Management: Full CRUD functionality for products inside the Brand Console. Each product record includes title, description, materials, sustainability badges, media (images, videos, AR in GLB/USDZ), and variant-level stock data. Firestore structure: /brands/{brandId}/products/{productId}. Includes: • Outlet Manager — move products to Outlet section with discount/timer logic (Cloud Function onOutletSchedule). • Promotions Engine — schedule-based discounts (/promos collection). • Sustainability Tagging — eco-label badges stored in /dictionaries/sustainability.
- Shopping and Checkout Flow: Complete end-to-end shopping experience: Home → Search → Product Detail (PDP) → Cart → Checkout → Order Confirmation. Supports multi-seller split shipments (brand/shop origin). Cart and order data stored in Firestore (/carts, /orders). Cloud Functions handle order validation, stock reservation, and payment webhooks (onOrderCreate, onPaymentWebhook). Integrations: Stripe/ЮKassa. Checkout includes “Frequently Bought Together” recommendations (Cloud Function recoAlsoBought) and loyalty point redemption.
- AI Stylist: AI-powered outfit suggestion engine leveraging catalog attributes (category, color, size, brand synergy). Accessible via a floating action button on all main screens. Modal chat interface presents curated looks, combining text and image cards. Powered by Cloud Function aiStylistSuggest, backed by Firestore item attributes and AI model (LLM endpoint). AI output supports “Add to Cart”, “Save Look”, and “Try in AR” actions.
- My Wardrobe and Lookboards: Personalized wardrobe view with AI-driven recommendations. Users can save owned items, create Lookboards, and share them publicly. Firestore collections: /users/{uid}/wardrobe, /lookboards/{uid}/looks. Includes AI Look Generation placeholder — blends wardrobe + catalog items via generative preview. AR Try-On supported with WebXR or USDZ viewer modal.
- B2B Linesheet and Pre-Order Management: Brands can create and distribute linesheets to partner shops. Linesheets contain wholesale pricing, MOQ, prepack data, and preview images. Firestore: /b2b/linesheets/{sheetId} and /b2b/preorders/{poId}. Pre-orders include status, negotiation chat, and approval workflow. Shops can request to add confirmed products to their Portfolio (requires brand approval). Cloud Function syncMoySklad provides optional ERP sync.
- Community Looks and AR Preview: Social layer for style expression and discovery. Supports user-, brand-, and shop-generated Looks: save, share, like, comment, and follow creators. Integrated with Firebase Storage for photo/video uploads and AR assets (GLB/USDZ). Includes AI render preview placeholder for generating outfits with mixed media. Firestore path: /community/looks/{lookId} with metadata and engagement metrics.
- Outlet Section: Dedicated catalog for past-season inventory marked by brands. Products automatically migrate based on age, season, or brand schedule. Features dynamic discount badges, countdown timers, and sustainability metrics. Client UI: same as catalog, but sorted by discount %. Firestore: /brands/{brandId}/products/{productId}.status.outlet = true.
- Brand Console: Dashboard for brand partners to manage catalog, promotions, analytics, and B2B operations. Key modules: • Product Management (CRUD + media upload). • Promos & Outlet Controls (schedule, discount). • Analytics (sales, views, conversion, returns, geo heatmap). • Content Publishing (news, lookbooks, articles). • B2B Linesheets & Preorders. • Integrations with ERP/WMS (via CSV or MoySklad API). Firestore collections: /brands, /brands/{brandId}/products, /b2b.
- Shop Console: B2B and retail interface for verified shops/buyers. Modules include: • Portfolio Management (approved products from brands). • Local Pricing & Stock Editing. • Orders and Pre-orders tracking. • Analytics Dashboard (sell-through, margins, brand performance). • Multi-user Roles (buyer, accountant, manager). • ERP/CSV Sync for stock and pricing. Firestore: /shops/{shopId} and /shops/{shopId}/portfolio.
- Admin Console: Central management interface for the entire platform. Modules: • Moderation (content, reviews, fraud). • Roles & Permissions Editor (RBAC toggle UI). • Dictionaries Management (categories, sizes, attributes, badges). • Bonus Program Configurator. • Featured Placement Manager (home banners, carousels). • Analytics & System Health (GMV, AOV, LTV, retention, performance logs). • Global Settings (tariffs, fees, SLA, notifications). Data powered by Firestore, Cloud Functions, GA4, and BigQuery.
- Push Notifications and Events: Powered by Firebase Cloud Messaging (FCM). Topics include: price_drop_{productId}, new_collection_{brandId}, live_event_{brandId}, order_status_{orderId}, b2b_updates_{brandId}_{shopId}. Triggered by Cloud Functions and user preferences in Firestore (/users/{uid}.preferences.notif). All analytics logged to GA4 and BigQuery.
- AI & AR Integration Framework: • AI — LLM-based stylist (aiStylistSuggest), recommendation engine (recoAlsoBought), and trend analysis placeholder (aiTrendPredict). • AR — Product and Look 3D preview (USDZ/GLB via Firebase Storage). Modal AR viewer integrated in PDP and Lookboards. • AI + AR fusion — generate AI outfit → preview → AR try-on (stubbed).
- Analytics and Reporting: All key user and system actions logged via Firebase Analytics and exported to BigQuery: events: view_item, add_to_cart, begin_checkout, purchase, follow_brand, create_look, ai_stylist_use, outlet_visit, b2b_preorder_create. Dashboards: • Clients — activity, retention, loyalty usage. • Brands — sales, conversion, returns. • Shops — sell-through, geo, margin. • Admin — GMV, cohorts, ROI, fraud flags.
- Design & UX Layer: Luxury-minimal interface with generous whitespace, neutral tones, and accent AI Blue. Consistent UI across devices (PWA + mobile app). Responsive grid, soft transitions, elevated typography. Key UX patterns: • Floating AI Stylist button (persistent). • Modular cards (product, look, event). • Contextual overlays (quick add, AR preview). • Loyalty & bonus widget on Home and Checkout. Accessibility: WCAG AA compliance, keyboard navigation.
- Legal and Compliance Framework: • GDPR / 152-ФЗ / DPA compliance for personal data. • E-contract templates for brands/shops (stored in /legal/contracts). • Content copyright validation for brand assets. • ESG and Sustainability Score integration. • Tokenized bonuses (if enabled) managed via /loyalty/tokens.
- System Architecture Overview: • Frontend: Next.js + Firebase Hosting (PWA-ready). • Backend: FastAPI (AI & payment webhooks) + Cloud Functions. • Database: Firestore + BigQuery. • Storage: Firebase Storage (images, videos, AR). • AI Layer: OpenAI GPT / Gemini API via callable functions. • Auth & Roles: Firebase Auth + Firestore. • Realtime: Firestore listeners + FCM updates. • Monitoring: Cloud Logging + Admin Dashboard.

## Style Guidelines:

- • Primary: White #FFFFFF — base for clarity and purity. • Secondary: Carbon Black #101114 — deep neutral providing contrast and luxury tone. • Accent: AI Blue #3E82F7 — used sparingly for interactive elements (CTAs, highlights). • Supporting: Taupe #C9B9A6 and Cream #F6F2EC — warm, elegant neutrals. The color palette ensures a refined, balanced contrast suitable for high-end fashion aesthetics.
- • Structure: Modular grid system with consistent spacing and a strong vertical rhythm. • Whitespace: Generous whitespace to ensure a clean, uncluttered, and premium user experience. • Alignment: Center-focused content blocks, ensuring visual calm and easy scanning. • Visual hierarchy: Hero areas emphasize imagery; secondary content fades softly into neutral backgrounds. • Responsiveness: Fully adaptive layout optimized for desktop, tablet, and mobile devices with fluid scaling of margins and padding.
- • Titles / Headings: 'Playfair Display' (serif) — elegant, timeless, and fashion-oriented; conveys editorial sophistication. *Note: currently only Google Fonts are supported.* • User Interface / Body Text: 'Inter' (sans-serif) — clean, modern, highly legible for UI labels and short text blocks. The app contains minimal long-form text, focusing on clarity and visual balance. • Scale: • H1 — 48px / bold / 1.2 line-height • H2 — 32px / semibold • Body — 16px / regular • Caption — 12px / medium • Contrast: Always meets WCAG AA standards for accessibility.
- • Style: Minimal, custom-drawn icons inspired by thin-line geometry. • Tone: Neutral monochrome (black or taupe), no color fills except for interaction states. • Functionality: Designed for clarity and fast recognition; uses consistent visual weight and corner radius. • Usage: • Outline icons for passive states • Filled or tinted icons for active states • Rounded edges to reflect brand softness and human touch
- • Motion Philosophy: “Quiet Interaction” — animation should feel like breathing, not flashing. • Transitions: Subtle fade-ins, opacity shifts, and smooth transforms (200–400ms easing). • Hover Effects: Soft scaling (1.03–1.05) with light shadow or color glow (AI Blue). • Loading States: Gentle pulsing gradients or skeleton screens with neutral tones. • Page Transitions: Fade + slide; maintain visual continuity rather than hard cuts. • AR / AI Elements: Slow luminance drift effect for intelligent modules (optional ambient shimmer).
