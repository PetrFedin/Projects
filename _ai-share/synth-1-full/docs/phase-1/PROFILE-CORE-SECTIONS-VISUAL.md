# Канон визуала: ядро профилей (этап 1)

Цель: в профилях оставляем только **действие → следующий шаг → done‑состояние**. Дубликаты (заголовки/описания/пояснения) запрещены.

## 1) Ядро профиля бренда (Organization)

Формат на странице раздела:
- **Название** (с маленькой иконкой)
- **1 строка описания**
- **Контент**: только блоки, которые двигают процесс (формы/таблицы/фильтры/CTA).  
  Если блок не влияет на завершение процесса — его нет.

### Секции (title / icon / desc / path)

- **Профиль бренда** / `Building2` / Юр.данные, контакты, Brand DNA. / `/brand`
- **Команда** / `Users` / Роли, доступы, задачи, активность. / `/brand/team`
- **Документооборот** / `FileText` / Договоры/акты/спеки/ЭДО. / `/brand/documents`
- **Интеграции** / `Zap` / ERP/маркетплейсы/логистика/webhooks. / `/brand/integrations`
- **Подписка и биллинг** / `CreditCard` / План, лимиты, платежи. / `/brand/subscription`
- **Безопасность** / `ShieldCheck` / 2FA, API keys, сессии, аудит. / `/brand/security`
- **Настройки** / `Settings` / Уведомления, локализация, налоги. / `/brand/settings`
- **ЭДО и маркировка** / `ShieldCheck` / ЧЗ, КИЗ, синхронизация. / `/brand/compliance`

## 2) Процессы (от начала до done)

Общий принцип: **одна точка истины** о статусе и одна «кнопка следующего шага».

### `/brand` — Профиль бренда
- **Start**: профиль открыт, виден блок обязательных полей
- **Action**: заполнить обязательные поля → сохранить
- **Done**: индикатор «профиль заполнен» (без доп. текста)

### `/brand/team` — Команда
- **Start**: список пуст/есть
- **Action**: пригласить → назначить роль/права → сохранить
- **Done**: есть минимум 1 ответственный (роль), приглашения без pending

### `/brand/documents` — Документооборот
- **Start**: список документов
- **Action**: создать/загрузить → назначить контрагент → отправить на подпись
- **Done**: документ в статусе «подписан» или «в работе» (валидный маршрут дальше)

### `/brand/integrations` — Интеграции
- **Start**: список интеграций + статусы
- **Action**: подключить → проверить статус/health
- **Done**: интеграция в `active` без ошибок

### `/brand/subscription` — Подписка и биллинг
- **Start**: текущий план + лимиты
- **Action**: выбрать план → оплатить/подтвердить
- **Done**: план активен, следующий платёж определён

### `/brand/security` — Безопасность
- **Start**: 2FA/API keys/сессии
- **Action**: включить 2FA → создать API key (при необходимости) → завершить
- **Done**: 2FA включена, ключи/сессии в ожидаемом состоянии

### `/brand/settings` — Настройки
- **Start**: параметры и уведомления
- **Action**: настроить → сохранить
- **Done**: нет «незаполнено/требует внимания» для критичных настроек

### `/brand/compliance` — ЭДО и маркировка
- **Start**: провайдер/сертификаты/статусы
- **Action**: подключить провайдера → загрузить/обновить сертификаты → проверить
- **Done**: статусы зелёные, нет истекающих критичных сертификатов

## 3) Запреты (чтобы не засорять UI)

- Не дублировать заголовок раздела внутри страницы, если он уже в layout.
- Не держать «инфо‑плашки» без CTA или без влияния на `Done`.
- Не повторять «см. также» в нескольких местах: максимум один блок связей внизу.
- Фильтры/кнопки — только если они меняют список/состояние (не декор).
