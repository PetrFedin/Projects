## 🔐 СИНХРОНИЗАЦИЯ С РОЛЯМИ И ПРОФИЛЯМИ

### **Источники данных**:

```typescript
// 1. User Context Hook
const userContext = useUserContext();
const { 
  isRetailer,     // ← Бренд определяет роль
  isBrand, 
  isBuyer, 
  currentOrg,     // ← Организация (название, лого, лимиты)
  user            // ← Профиль (имя, email, роли)
} = userContext;

// 2. Primary Role Detection
const primaryRole = useMemo((): B2BUserRole => {
  if (isRetailer) return 'retailer';
  if (isBrand) return 'brand';
  if (isBuyer) return 'buyer';
  return 'buyer'; // default
}, [isRetailer, isBrand, isBuyer]);

// 3. Фильтрация функций
const allowedItems = ROLE_PERMISSIONS[primaryRole];
const filteredItems = WORKSPACE_ITEMS
  .filter(item => item.category === activeTab)
  .filter(item => allowedItems.includes(item.id));
```

### **Связи с профилями**:

| Профильные данные | Где используется |
|-------------------|------------------|
| `user.displayName` | Role Context Card (имя пользователя) |
| `user.roles[]` | Primary role detection |
| `currentOrg.name` | Role Context Card (название организации) |
| `currentOrg.creditLine` | Метрики в Payment Hub |
| `currentOrg.loyaltyTier` | Метрики в Loyalty Status |

## 🎯 ПРЕИМУЩЕСТВА НОВОЙ СТРУКТУРЫ

### **1. Data-Driven Architecture**
```
Вместо:  Хардкод 14 виджетов в JSX
Теперь:  Один WORKSPACE_ITEMS массив
```
**Результат**: Легко добавить/удалить/изменить функцию

---

### **2. Single Source of Truth**
```
Вместо:  Дублирование условий {isRetailer && ...}
Теперь:  Одна матрица ROLE_PERMISSIONS
```
**Результат**: Нет ошибок рассинхронизации

---

### **3. Role-Based Персонализация**
```
Вместо:  Все видят все виджеты с условиями
Теперь:  Каждый видит только свои функции
```
**Результат**: Нет cognitive overload

---

### **4. Visual Clarity**
```
Вместо:  Плоский список виджетов
Теперь:  Матрица с color coding, приоритетами, метриками
```
**Результат**: Мгновенное понимание важности

---

### **5. Transparency**
```
Вместо:  Непонятно кто что может
Теперь:  Легенда ролей показывает ВСЕ права
```
**Результат**: Прозрачная система доступов

---

### **6. Scalability**
```
Вместо:  Добавить роль = править 20 мест
Теперь:  Добавить роль = 3 строки в ROLE_CONFIG
```
**Результат**: Быстрое масштабирование

---

### **7. Maintenance**
```
Вместо:  1,151 строк в одном файле
Теперь:  280 (данные) + 423 (UI) = 703 строки
```
**Результат**: Легче поддерживать и дебажить
