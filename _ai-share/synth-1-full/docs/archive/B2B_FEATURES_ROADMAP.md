# Функции и опции JOOR / NuOrder / Fashion Cloud / Colect / Zedonk / SparkLayer — что внедрить

## Уже в проекте (кратко)

| Платформа     | Реализовано |
|---------------|-------------|
| **JOOR**      | Окна доставки, MOQ, mark order exported, re-export, getJoorConfigFromEnv, экспорт заказа через API |
| **NuOrder**   | OAuth 1.0, company codes, order/new (создание заказа), API export-order, company-codes |
| **Fashion Cloud** | Products, stock, catalog status, sync products, API catalog-summary, блок на RFQ |
| **SparkLayer**| Price lists, pricing by SKU, calculate pricing, API price-lists, блок на «Условия по коллекциям» |
| **Colect**    | Статус в виджете, навигация (лукбуки/проекты) |
| **Zedonk**    | Статус в виджете, навигация (агентский кабинет) |

---

## JOOR — что взять и внедрить

| Функция | Описание | Приоритет | Куда в платформе |
|---------|----------|-----------|------------------|
| **Inventory (v2)** | Отправка остатков: по складам, датам доступности, case pack. Overwrite или только изменения. | Высокий | Склад/остатки бренда → синхрон с JOOR при сохранении или по кнопке «Синхронизировать с JOOR» |
| **Prices (v4)** | Цены по SKU, несколько типов (wholesale/retail), по валютам и по клиентам. | Высокий | Финансы/прайсы бренда, матрица заказа магазина — подтягивать цены из JOOR при наличии токена |
| **Styles / Products** | Создание/обновление/удаление стилей, привязка к лайншитам, категории, кастомные поля, изображения. | Средний | PIM/коллекции → «Экспорт в JOOR» (стили, цвета, размеры) |
| **Customers** | Sales rep, склады, скидки по заказу/позиции, «двери» по адресам доставки, bulk create/update. | Средний | Партнёры/байеры → синхрон контактов и правил с JOOR |
| **Orders (import)** | Импорт заказов из JOOR в платформу: скидки, налоги, даты, обновление существующих. | Высокий | B2B заказы бренда — фоновая синхронизация или webhook «новый заказ в JOOR» |

**Рекомендация по приоритету:** сначала **Inventory** и **Prices** (сильно влияют на заказы и каталог), затем **Orders import**, потом Styles и Customers.

## NuOrder — что взять и внедрить

| Функция | Описание | Приоритет | Куда в платформе |
|---------|----------|-----------|------------------|
| **Orders Shipments** | Отправка статусов отгрузок в NuOrder (ERP → NuOrder). | Высокий | Отгрузки (ASN) бренда — при смене статуса отправлять в NuOrder |
| **Orders Edits** | Обновление заказов (изменения позиций, отмена строк). | Высокий | Заявки на изменение заказа (order amendments) — дублировать в NuOrder |
| **Inventory** | Синхрон остатков в NuOrder (pre-book, ATS). | Высокий | Склад бренда → «Синхронизировать остатки с NuOrder» |
| **Replenishment** | Данные по пополнению для байеров. | Средний | Магазин B2B — блок «Replenishment» или в календаре поставок |

**Рекомендация:** после текущего order/new добавить **Inventory** и **Orders Shipments**; затем **Orders Edits** и при необходимости Replenishment.

## Fashion Cloud — что взять и внедрить

| Функция | Описание | Приоритет | Куда в платформе |
|---------|----------|-----------|------------------|
| **Orders import / draft orders** | Импорт заказов и черновиков из внешних систем, webhooks по заказам. | Высокий | B2B заказы — «Импорт из Fashion Cloud» или webhook в API |
| **Stock bulk upsert** | Массовое обновление остатков (bulk). | Высокий | Склад/остатки — кнопка «Выгрузить остатки в Fashion Cloud» |
| **Catalog options + media** | Управление опциями (продукты), цены, размеры, медиа (фото, видео, 3D). | Средний | PIM/медиа — синхрон карточек и медиа в Fashion Cloud |
| **Brands, divisions, seasons, drops** | Справочники брендов, дивизионов, сезонов, дропов. | Низкий | Справочники платформы или настройки синдикации |

**Рекомендация:** сначала **Orders import** (или webhook) и **Stock bulk upsert**, затем расширить каталог (options + media).

## SparkLayer — что взять и внедрить

| Функция | Описание | Приоритет | Куда в платформе |
|---------|----------|-----------|------------------|
| **Core API: products, customers, orders** | Продукты, клиенты, заказы (REST + OAuth2). | Высокий | Единый слой: при наличии SparkLayer — подтягивать клиентов и заказы, пушить продукты |
| **Quoting** | Квоты, правки, срок действия. | Средний | Магазин B2B — «Запрос коммерческого предложения» с отправкой в SparkLayer |
| **Customer rules / discounts** | Правила по клиентам, скидки. | Средний | Партнёры/прайсы — отображение правил и скидок из SparkLayer |
| **JavaScript SDK (frontend)** | Виджеты корзины/чекаута B2B на сайте. | Средний | Магазин/лендинг — встраивание SparkLayer checkout при необходимости |

**Рекомендация:** после текущего Pricing API подключить **Core API** (products, customers, orders); затем Quoting и customer rules.

## Colect — что взять и внедрить

| Функция | Описание | Приоритет | Куда в платформе |
|---------|----------|-----------|------------------|
| **Lookbook structure** | Главы, Key Looks, связь с заказом (order grid). | Высокий | Лукбуки/проекты бренда — структура «главы + ключевые луки», добавление в заказ из лукбука |
| **Content: фото, видео, 3D** | Высокое разрешение, видео, 3D-модели. | Средний | Медиа/DAM — экспорт в формат Colect или ссылки для Colect |
| **Режимы показа** | Sales App, Brand Portal, Digital Showroom, Hybrid (Welcome/Storytelling/Selling). | Низкий | Настройки шоурума/лукбука — выбор режима отображения (если будет связка с Colect) |

**Ограничение:** публичного API в открытом доступе нет; интеграция возможна по запросу к Colect. В проекте пока — константы/типы и навигация; при появлении API — добавить клиент по аналогии с JOOR/NuOrder.

## Zedonk — что взять и внедрить

| Функция | Описание | Приоритет | Куда в платформе |
|---------|----------|-----------|------------------|
| **Приём заказов из JOOR/NuOrder** | Заказы с B2B-платформ в единый порядок. | Высокий | Уже в духе платформы: «Заказы» агрегируют источники; при появлении Zedonk API — приём заказов из Zedonk |
| **Синхрон стилей, лайншитов, клиентов, остатков с JOOR** | Как в коннекторе JOOR–Zedonk. | Средний | При наличии Zedonk API — дублировать синхрон стилей/каталога/остатков (или делегировать JOOR) |
| **Multi-brand / agent** | Один агент, несколько брендов, сводные заказы. | Высокий | Агентский кабинет магазина (Zedonk-style) — уже есть навигация; при API — подтягивать бренды и сводные заказы |

**Ограничение:** отдельного публичного API для «агентского мультибренда» в поиске не видно; интеграция через партнёрские каналы (JOOR и др.). В проекте — статус и навигация; при появлении API — клиент и точки синхронизации.

## Сводная очередность внедрения

1. **JOOR:** Inventory (v2), Prices (v4), импорт заказов из JOOR.  
2. **NuOrder:** Inventory, Orders Shipments, Orders Edits.  
3. **Fashion Cloud:** Orders import / webhook, Stock bulk upsert.  
4. **SparkLayer:** Core API (products, customers, orders).  
5. **Colect:** при появлении API — lookbook structure и контент.  
6. **Zedonk:** при появлении API — приём заказов и мультибренд/агент.

Если нужно начать с малого: первым делом реализовать **JOOR Inventory** или **NuOrder Inventory** + **Orders Shipments** — они сразу усиливают текущие сценарии заказов и склада.
