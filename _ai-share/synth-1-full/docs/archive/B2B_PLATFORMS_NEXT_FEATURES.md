# JOOR / NuOrder / Fashion Cloud / Colect / Zedonk / SparkLayer — что уже есть и что забирать дальше

Обзор по платформам: что уже встроено в проект и какие фичи логично добавить или усилить следующими.

---

## JOOR

### Уже в проекте
- Виртуальные выставки (trade-shows), Passport выставки (brand + shop).
- Заявки байеров на партнёрство (buyer-applications) с апрувом брендом и store.
- Многошаговое согласование заказа (order-approval-workflow): байер → менеджер магазина → бренд.
- Заметки к заказу (в карточке заказа), внутренние заметки бренда.
- Окна доставки (дропы), MOQ (joor-constants), ASN/отгрузки по заказу.
- Трекинг отгрузки (блок «Отгрузка и доставка», баннер «Заказ отгружен»).
- Финансы партнёра (лимит, оплачено, JOOR Pay — ссылки и экран).
- Написание заказа по коллекции, матрица заказа, режим заказа (order-mode).

### Забирать / улучшать дальше
| Фича | Описание | Приоритет |
|------|----------|-----------|
| **JOOR Pay (embedded payment)** | Реальный флоу оплаты внутри платформы (сейчас только ссылка на оплату). Интеграция с экраном «Оплата заказов» и статусами «оплачен». | **P0** |
| **Engagement / визиты** | Страница engagement есть; добавить: визиты шоурума/лайншита, открытия каталога, активность по партнёрам (дашборд «кто смотрел что»). | P1 |
| **PO (Purchase Order)** | Страница po есть; усилить: генерация/загрузка PO, привязка к заказу, нумерация и экспорт в PDF. | P1 |
| **Credit limit в реальном времени** | Показ лимита и «использовано» в матрице заказа и при отправке на согласование; блокировка при превышении (частично есть в WholesaleOrderMatrix). | P1 |

## NuOrder

### Уже в проекте
- Режим заказа (Buy Now / Reorder / Pre-order), матрица заказа, быстрый заказ по артикулам, reorder из истории (с sell-through мок).
- Кампании по лайншитам (linesheet-campaigns): отправка байерам, метрики открытий/кликов/заказов.
- Версии лайншита (linesheet-versions): Early Bird, VIP, Outlet, Stock Lot.
- Календарь поставок / drop dates (delivery-calendar), Working Order (экспорт/импорт Excel), EZ Order.
- Аналитика по заказам (order-analytics): топ стилей, тренды.
- Права по территории/каналу (buyer-rights), ATS (available to sell) в матрице.

### Забирать / улучшать дальше
| Фича | Описание | Приоритет |
|------|----------|-----------|
| **Планирование ассортимента до заказа** | Страница assortment-planning есть; углубить: план по категориям/брендам, сравнение с прошлым сезоном, рекомендации по глубине ассортимента. | **P0** |
| **Календарь поставок (drop dates)** | Страница delivery-calendar есть; добавить: визуальный календарь по брендам/коллекциям, дедлайны заказа по дропу, напоминания. | P1 |
| **Трекинг кампаний** | В linesheet-campaigns уже open/click/order rate; добавить: ссылки «кто открыл», «кто заказал», экспорт в CSV. | P1 |
| **Working Order — импорт** | Сейчас в основном экспорт; добавить: загрузка Excel по шаблону → разбор строк и создание/обновление драфта заказа. | P2 |

## Fashion Cloud

### Уже в проекте
- Синдикация контента (content-syndication): расписание выгрузки, валидация контракта B2B (размерная сетка, состав, уход, EAN, медиа), лог выгрузки и SKU с ошибками.
- B2B каталог байера (shop/b2b/catalog): каталог из PIM с атрибутами по контракту.
- Контракт атрибутов (b2b-catalog-contract), каналы контента по продукту (content-syndication.ts).

### Забирать / улучшать дальше
| Фича | Описание | Приоритет |
|------|----------|-----------|
| **Валидация перед выгрузкой** | Блок «SKU с ошибками» и отчёт; доработать: список полей с проблемами по каждому SKU, экспорт списка на доработку, статус «готов к выгрузке». | **P0** |
| **Расписание и логи выгрузок** | История выгрузок по дате, статус (успех/ошибка), объём (сколько SKU), ссылка на лог ошибок. | P1 |
| **Поддержка нескольких каналов** | Разные «каналы» (ритейл A/B, маркетплейс, аутлет) с разными наборами атрибутов и правилами валидации. | P2 |

## Colect

### Уже в проекте
- Лукбуки как проекты (lookbook-projects): права (кто видит, до какой даты), watermarked PDF, заказ из лукбука.
- Карта партнёров по территориям (partner-map): кто где торгует, конфликты, эксклюзив.
- Шаринг лукбука (lookbook-share): ссылка с истечением срока, пароль.
- Лукбуки для байера (shop/b2b/lookbooks): доступ по правам и дате, просмотр PDF, заказ из лукбука.
- Комментарий по строке заказа (Colect/JOOR) в карточке заказа.

### Забирать / улучшать дальше
| Фича | Описание | Приоритет |
|------|----------|-----------|
| **Права на лукбук по партнёру/территории** | Управление «кто видит какой лукбук до какой даты» списком; связь с partner-map (территория → доступ к дропу). | **P0** |
| **Watermarked PDF по проекту** | Генерация или загрузка PDF с водяным знаком (имя партнёра/дата); скачивание из проекта. | P1 |
| **Конфликты территории на карте** | В partner-map: визуально подсветить пересечения территорий и эксклюзивные зоны; предупреждения при назначении нового партнёра. | P1 |
| **Заказ «из лукбука»** | Флоу: просмотр лукбука → выбор луков/стилей → добавление в корзину заказа (связь с матрицей или быстрым заказом). | P2 |

## Zedonk

- Сводный заказ агента (agent/consolidated-order): один драфт с позициями по разным брендам, MOV/MOQ по бренду.
- Логика consolidated-order-draft, agent-context.

### Забирать / улучшать дальше
| Фича | Описание | Приоритет |
|------|----------|-----------|
| **Комиссии агента по брендам** | Дашборд: комиссия по сделке/заказу по каждому бренду, накопление за период, отчёт для выгрузки. | **P0** |

## SparkLayer

### Уже в проекте
- Прайс-листы и акции (price-lists): разные уровни цен по периоду/каналу (Retail A/B, Outlet), множитель к цене.
- Правила по заказу (order-rules): MOQ, MOV, территориальные ограничения; префлайт в матрице (кредит, MOV, территория).
- Упоминания в WholesaleOrderMatrix: ценовой уровень из URL/партнёра, Pre-flight check, блокировка по лимиту.

### Забирать / улучшать дальше
| Фича | Описание | Приоритет |
|------|----------|-----------|
| **Прайс-листы по каналам с датами** | В price-lists: период действия (с/по), привязка к каналу (ритейл, аутлет, маркетплейс); применение в матрице при выборе заказа/партнёра. | **P0** |
| **Акции и промо по периоду** | Отдельный блок «Акции»: скидка % или фикс. цена на группу SKU/коллекцию на период; отображение в матрице и в итоге заказа. | P1 |
| **MOV/MOQ по партнёру и по бренду** | Уже есть частично; централизовать: настройки MOV/MOQ по контракту партнёра и по бренду, показ в одном месте и в матрице. | P1 |
| **Блокировка строк с причиной** | В матрице: при нарушении территории/MOV/MOQ/кредита — не только предупреждение, но и блокировка строки с текстом причины (уже заложено в order-rules). | P2 |

## Сводная очередность (что делать первым)

1. **JOOR:** встроенный платёж (JOOR Pay) и учёт кредитного лимита в реальном времени.
2. **SparkLayer:** прайс-листы по каналам и датам + акции по периоду; связь с матрицей.
3. **NuOrder:** углубление планирования ассортимента и календаря поставок.
4. **Colect:** права на лукбук по партнёру/дате и заказ из лукбука.
5. **Zedonk:** комиссии агента по брендам и отправка сводного заказа по брендам.
6. **Fashion Cloud:** валидация и отчёты по SKU с ошибками, логи выгрузок.

Маршруты и ссылки для перечисленных модулей уже есть в `src/lib/routes.ts` и `src/lib/data/entity-links.ts` — остаётся наращивать логику и UI на существующих страницах или добавлять дочерние экраны.
