# Структура навигации личных кабинетов

## Общие принципы

Навигация в личных кабинетах (Brand, Shop, Admin) организована по принципам JOOR/NuOrder:

1. **Двухуровневая структура**: Группы разделов → Разделы → Подразделы
2. **Нет дублирования**: Каждая страница имеет только одну панель навигации в layout
3. **Логическая группировка**: Разделы сгруппированы по бизнес-логике (B2B, B2C, Производство и т.д.)
4. **Единообразие**: Все кабинеты используют одинаковую структуру компонентов

## Архитектура

### Layout структура

```
┌─────────────────────────────────────────────────────┐
│ Header (Логотип, название кабинета, группы разделов)│
├─────────────────────────────────────────────────────┤
│ Заголовок группы                    │ Текущий раздел│
├─────────────────────────────────────────────────────┤
│ [Раздел 1] [Раздел 2] [Раздел 3] ...               │ ← Основные разделы
├─────────────────────────────────────────────────────┤
│ [Подраздел 1] [Подраздел 2] ...                    │ ← Подразделы (если есть)
├─────────────────────────────────────────────────────┤
│                                                     │
│ Контент страницы                                    │
│                                                     │
└─────────────────────────────────────────────────────┘
```

### Файлы навигации

- **Shop**: `/src/lib/data/shop-navigation-normalized.ts`

## Правила для разработчиков

### ✅ Правильно

1. **Одна панель навигации** - только в layout, не дублировать на страницах
2. **Функциональные табы** - можно использовать внутри страниц для переключения контента (например, "Стримы/Видео/Сторис" на странице медиа)
3. **Подразделы в навигации** - использовать для фильтров и статусов (например, "Все заказы/Черновики/Подтвержденные")
4. **Логическая группировка** - группировать по бизнес-логике, не по техническим признакам

### ❌ Неправильно

1. **Дублирующие панели** - не создавать дополнительные TabsList в layout страниц
2. **Вложенные layouts** - не создавать layout.tsx внутри подпапок (например, `/brand/inventory/layout.tsx`)
3. **Разрозненные разделы** - не размещать связанные функции в разных группах
4. **Технические группировки** - не группировать по типу данных вместо бизнес-процессов

## Добавление новых разделов

### 1. Добавить в навигационный файл

```typescript
// В соответствующий файл (brand-navigation.ts, shop-navigation-normalized.ts, admin-navigation-normalized.ts)
{
    href: '/brand/new-section',
    value: 'new-section',
    label: 'Новый раздел',
    icon: IconComponent,
    description: 'Описание раздела',
    subsections: [ // Опционально
        { href: '/brand/new-section', label: 'Подраздел 1', value: 'sub1' },
        { href: '/brand/new-section?filter=active', label: 'Подраздел 2', value: 'sub2' }
    ]
}
```

### 2. Создать страницу

```typescript
// /app/brand/new-section/page.tsx
export default function NewSectionPage() {
    return (
        <div className="space-y-6">
            {/* Контент без дополнительных панелей навигации */}
        </div>
    );
}
```

### 3. Не создавать layout.tsx

Layout уже есть на уровне кабинета (`/app/brand/layout.tsx`), дополнительный не нужен.

## Миграция существующих страниц

Если страница имеет дублирующую навигацию:

1. Удалить `layout.tsx` из подпапки
2. Удалить `<Tabs>` и `<TabsList>` со страницы (если они дублируют основную навигацию)
3. Оставить только функциональные табы (если они переключают контент внутри страницы)
4. Убедиться, что подразделы добавлены в навигационный файл

## Вспомогательные функции

```typescript
// Получить подразделы для раздела
import { getSubsections } from '@/lib/data/brand-navigation';
const subsections = getSubsections('products');

// Найти конкретный подраздел
import { findSubsection } from '@/lib/data/brand-navigation';
const subsection = findSubsection('products', 'active');
```

## Стилизация

Все кабинеты используют единый стиль:

- **Группы разделов**: Горизонтальная панель с кнопками, активная группа выделена темным фоном
- **Основные разделы**: Кнопки с иконками, активный раздел с цветной рамкой
- **Подразделы**: Компактные кнопки, активный подраздел с цветным фоном
- **Цветовая схема**: 
  - Brand: indigo
  - Shop: rose
  - Admin: indigo

## Примеры использования

### Страница с подразделами

```typescript
// Навигация автоматически отображает подразделы из brand-navigation.ts
// Страница просто рендерит контент
export default function ProductsPage() {
    const searchParams = useSearchParams();
    const filter = searchParams.get('filter') || 'all';
    
    return (
        <div className="space-y-6">
            {/* Контент в зависимости от filter */}
        </div>
    );
}
```

### Страница с функциональными табами

```typescript
// Функциональные табы для переключения контента - это правильно
export default function MediaPage() {
    const [activeTab, setActiveTab] = useState('streams');
    
    return (
        <div className="space-y-6">
            <Tabs value={activeTab} onValueChange={setActiveTab}>
                <TabsList>
                    <TabsTrigger value="streams">Стримы</TabsTrigger>
                    <TabsTrigger value="videos">Видео</TabsTrigger>
                    <TabsTrigger value="stories">Сторис</TabsTrigger>
                </TabsList>
                <TabsContent value="streams">{/* Контент */}</TabsContent>
                <TabsContent value="videos">{/* Контент */}</TabsContent>
                <TabsContent value="stories">{/* Контент */}</TabsContent>
            </Tabs>
        </div>
    );
}
```
