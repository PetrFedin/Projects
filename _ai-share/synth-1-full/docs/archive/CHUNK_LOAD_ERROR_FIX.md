# Исправление ChunkLoadError

## Проблемы

### 1. Страница без стилей (белая/«сырая» вёрстка)
Если контент виден, но без оформления (нет отступов, цветов, всё идёт в одну строку) — скорее всего CSS не загружается (404 на `/_next/static/css/...`). **Решение:** очистить кеш Next и обновить страницу с очисткой кеша браузера (см. п. 2–3 ниже).

### 2. ChunkLoadError
`ChunkLoadError: Loading chunk app/loyalty/page failed` - ошибка загрузки чанка в Next.js

## Решения (выполнить по порядку)

### 1. ✅ Исправлена конфигурация Next.js
- Добавлена webpack конфигурация для dev режима
- `output: 'standalone'` теперь только для production

### 2. Очистить кеш браузера
**В браузере:**
- Откройте DevTools (F12)
- Правый клик по кнопке обновления страницы
- Выберите "Очистить кеш и жёсткая перезагрузка" (Empty Cache and Hard Reload)

Или:
- Ctrl+Shift+R (Windows/Linux)
- Cmd+Shift+R (Mac)

### 3. Очистить Next.js кеш
```bash
cd /Users/petr/Projects/synth-1
rm -rf .next
npm run dev
```

### 4. Перезапустить dev сервер
```bash
# Остановите текущий сервер (Ctrl+C)
# Запустите заново
npm run dev
```

### 5. Если не помогает - очистить node_modules
```bash
rm -rf node_modules .next
npm install
npm run dev
```

## Причина проблемы
ChunkLoadError возникает когда:
- Dev сервер перезагрузился, но браузер пытается загрузить старые чанки
- Кеш браузера содержит устаревшие ссылки на файлы
- `output: 'standalone'` в dev режиме может вызывать проблемы

## Профилактика
- Регулярно очищайте кеш браузера при разработке
- Используйте Hard Reload при странных ошибках
- Избегайте `output: 'standalone'` в dev режиме
