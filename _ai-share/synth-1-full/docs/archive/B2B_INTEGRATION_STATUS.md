# Статус интеграций B2B (Fashion Cloud, NuORDER, SparkLayer, JOOR, Colect/Zedonk)

## 1. Fashion Cloud — контент и синдикация ✅
- **Сделано:** Единый PIM → каталог байера (ассортимент, медиа, атрибуты). Слой `content-syndication.ts`, страница бренда Content Syndication, синхронизация по расписанию (мок), индикатор «последнее обновление». Связка с Discover и карточкой товара в матрице (ссылка «В матрицу» с brand/sku).
- **Осталось:** При открытии матрицы по ссылке с `brand`/`sku` — подстановка строк из каталога (опционально).

---

## 2. NuORDER — инвентарь и «доступно к продаже» ⚠️
- **Сделано:** В матрице заказа отображается «ATS: 42» (захардкожено). Есть раздел ATS в b2b-workspace-matrix, ссылка на каталог.
- **Не сделано:**
  - ATS/сток в данных (по SKU/размеру/слоту) и отображение в матрице и Quick Order из одного источника.
  - Предупреждение «недостаточно остатка» при вводе qty > ATS.
  - Явная связь с картой остатков / B2B каталогом (единый источник правды).

---

## 3. SparkLayer — сегменты и прайсы ⚠️
- **Сделано:** На карточке партнёра указан «ценовой уровень: Standard B2B», кредитный лимит и блокировка заказа при исчерпании лимита. Есть Credit Risk, версии лайншита (Early Bird / VIP / Outlet).
- **Не сделано:**
  - Ценовые уровни (Retail A / B / Outlet) в данных и разные цены в матрице в зависимости от уровня.
  - Минималка по заказу/бренду в правилах и проверка при отправке.
  - Предупреждение до отправки заказа при приближении/превышении кредита (сейчас блок только при 0).

---

## 4. JOOR — выставки и визиты ✅ (частично)
- **Сделано:** Trade show (список, API), Passport выставки (бренд + байер), заявки байеров. **Привязка заказа к событию:** на странице «Написание заказа» (create-order) добавлен выбор «Привязать к событию» (FW26/SS26), параметр eventId передаётся в матрицу; из Passport байера ссылка «Создать заказ с выставки» ведёт на create-order?eventId=fw26. **Engagement:** дашборд бренда дополнен колонками «Лукбук» (открытия) и «Время на коллекции» (мин).
- **Осталось:** В карточке заказа и в списке заказов показывать «Событие: FW26» при наличии tradeShowId; инвайты/RSVP в UI.

---

## 5. Colect / Zedonk — территории и эксклюзивы ✅ (частично)
- **Сделано:** Territory Protection (правила по регионам), страница territory. **Карточка партнёра:** блок «Территория: по договору. Эксклюзив по региону/каналу настраивается брендом». **Конфликт при заявке:** на странице «Подать заявку» при указании страны/города показывается предупреждение «в вашем регионе уже есть эксклюзивный партнёр» (мок: Москва/ЦФО, СПб/СЗФО) с именем партнёра.
- **Осталось:** Отдельная страница «карта партнёров» с визуализацией территорий.

---

## 6. NuORDER / JOOR — финансы заказа ✅ (частично)
- **Сделано:** Оплата по этапам в Embedded Payment. **График платежей + инвойс:** на странице заказа бренда (b2b-orders/[orderId]) добавлен блок «График платежей и инвойс» — депозит / баланс / инвойс в одной временной шкале со статусами. **Экспорт в ERP:** в «Мои заказы» (shop) две кнопки: «CSV заказов» (как раньше) и «CSV строки+размеры» — один файл с колонками № строки, Артикул, Размер, Кол-во, Цена за ед., Сумма строки.
- **Осталось:** Подставить реальные строки заказа в экспорт (сейчас мок одна строка на заказ).

---

## 7. Zedonk — агентства / мульти-бренд ✅ (частично)
- **Сделано:** Роль `distributor_agent`, Sub-Agent Commission. **Мульти-бренд:** на странице «Написание заказа» выбор бренда трактуется как контекст агента (несколько брендов в одном кабинете). **Комиссия по строке/заказу:** в типе CommissionRecord добавлено поле commissionType: 'per_order' | 'per_line', в моках и на странице комиссий отображается «(по заказу)» / «(по строке)».
- **Осталось:** Переключатель бренда в шапке для роли агента (глобальный контекст).

---

## Порядок доработок (чтобы всё работало вертикально и горизонтально)

1. **NuORDER ATS** — слой ats/inventory, использование в матрице и Quick Order, предупреждение по остатку.
2. **SparkLayer** — ценовые уровни в типах и матрице, предупреждение по кредиту до отправки.
3. **JOOR** — order.tradeShowId в UI, связь Passport ↔ заказы; engagement: лукбук + время на коллекции.
4. **Territory** — территория на карточке партнёра, проверка эксклюзива при заявке.
5. **Финансы** — компонент «график платежей + инвойс», экспорт ERP (строки + размеры).
6. **Агент** — переключение бренда в кабинете агента, комиссия по строке/заказу.
