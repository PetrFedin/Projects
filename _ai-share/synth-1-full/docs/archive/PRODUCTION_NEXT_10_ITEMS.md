# Следующие 10 пунктов для интеграции в производство

После внедрения приоритетов (API/БД, роли, мобильная QC, SLA) — следующие шаги:

---

## 1. **Полная замена моков на API**
- Подключить `useProductionData` к `/api/production/*` для collections, skus, samples, orders
- Создать API routes: `/api/production/skus`, `/api/production/samples`, `/api/production/orders`, `/api/production/budget`
- Реализовать `POST`/`PATCH` для создания и обновления

## 2. **БД и Supabase/PostgreSQL**
- Схема таблиц: `collections`, `skus`, `sample_statuses`, `production_orders`, `audit_log`, `sla_config`
- Миграции и индексы
- Связь с `user` для ролей и `organization` для мультитенантности

## 3. **Расширенная аналитика**
- KPI-дашборд с on-time delivery %, качество по фабрикам, план vs факт
- Drill-down по коллекциям, периодам, фабрикам
- Экспорт отчётов в Excel/PDF

## 4. **Реальные интеграции PLM**
- Настройка подключения Gerber / CLO3D / Lectra
- Импорт BOM, градаций, лекал
- Синхронизация изменений

## 5. **Интеграция ERP/1С**
- Синхронизация заказов, остатков, финансов
- API для двустороннего обмена
- Маппинг полей и справочников

## 6. **Кабинет фабрики**
- Отдельный роут `/factory` с видом только своих PO
- Обновление статусов (раскрой, пошив, отгрузка)
- Загрузка QC-чеклистов и фото

## 7. **Email и Push-уведомления**
- Отправка при просрочке SLA, новом QC, amendment PO
- Настройка каналов (email, push) в триггерах
- Интеграция с провайдером (SendGrid, Firebase Cloud Messaging)

## 8. **Офлайн и PWA**
- Service Worker для кэширования
- Очередь действий при офлайне
- Синхронизация при восстановлении связи

## 9. **Тёмная тема**
- CSS-переменные для тёмного режима
- Переключатель в настройках
- Сохранение предпочтения в localStorage

## 10. **Расширенный аудит и история**
- Версионность BOM с diff
- История изменений PO (amendments) с деталями
- Фильтры по пользователю, сущности, дате
- Экспорт логов для compliance
