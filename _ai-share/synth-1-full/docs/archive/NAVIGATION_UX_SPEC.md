# UX-схема навигации Brand Center

## 1. Цели и принципы

- **Ничего не выезжает за границы** — все уровни меню вписываются в viewport
- **Меню открывается поверх контента** — overlay с затемнением фона
- **Простое управление** — один тап/клик для раскрытия, лёгкое закрытие
- **Адаптивность** — одна логика для desktop, tablet, mobile с разными точками входа

---

## 2. Структура уровней

```
Уровень 0: Группы (Организация | Логистика | Центр управления | ...)
    └── Уровень 1: Ссылки группы (Профиль и команда | Интеграции | Документооборот | ...)
            └── Уровень 2: Подразделы (если есть subsections)
```

**Пример:** Организация → Интеграции → [Маркетплейс | ERP/PLM/POS | Webhooks & API | SSO]

---

## 3. Desktop (≥1024px)

### 3.1 Точка входа
- Горизонтальная панель под KPI: группы как табы/пиллы
- Вторая строка: ссылки активной группы + подразделы (если есть)

### 3.2 Раскрытие уровней
- **Уровень 0 → 1:** Группа — клик переключает группу и показывает её ссылки во второй строке
- **Уровень 1 → 2 (если есть subsections):** при hover или клике на пункт с subsections:
  - **Вариант A (рекомендуемый):** Dropdown поверх контента
    - Выравнивание: левый край dropdown = левый край пункта
    - Позиция: `position: fixed`, расчёт через `getBoundingClientRect()` + `offsetParent`
    - Ограничение: не выходить за правый край viewport; при нехватке места — сдвиг влево
  - **Вариант B:** Подменю выезжает вправо от панели (каскадное)

### 3.3 Overlay-поведение
- При открытом dropdown/подменю:
  - Полупрозрачный backdrop (`rgba(0,0,0,0.2)`) на весь экран
  - `z-index: 50` для overlay, `z-index: 51` для меню
  - Клик по backdrop или Escape — закрытие
  - Меню не выходит за `window.innerWidth` / `window.innerHeight`

### 3.4 Overflow-правила
- Группы: горизонтальный скролл (`overflow-x: auto`), `scrollbar-hide`, `-webkit-overflow-scrolling: touch`
- Ссылки группы: то же
- Dropdown подразделов: `max-height: calc(100vh - 120px)`, внутренний скролл при длинном списке

## 4. Tablet (768px–1023px)

### 4.1 Точка входа
- Кнопка **«Меню»** (hamburger) в левом верхнем углу шапки
- При клике — **полноэкранный overlay** или **боковая панель справа** (ширина 320px)

### 4.2 Боковая панель (рекомендуемый вариант)
- Панель выезжает справа, `transform: translateX(0)` при открытии
- Ширина: `min(320px, 85vw)`
- Содержимое:
  - Группы — аккордеон (одна группа раскрыта)
  - Внутри группы — список ссылок
  - Ссылка с subsections — вложенный аккордеон или список с отступом

### 4.3 Раскрытие уровней
- Группа: клик раскрывает/сворачивает список ссылок (аккордеон)
- Ссылка с subsections: клик раскрывает подпункты под ней (вложенный аккордеон)
- Высота панели: `100vh`, внутренний скролл для длинного списка

### 4.4 Закрытие
- Кнопка ✕ в шапке панели
- Клик по backdrop (затемнённая область слева от панели)
- Свайп вправо (жест закрытия)
- Escape

---

## 5. Mobile (<768px)

### 5.1 Точка входа
- Кнопка **«Меню»** в шапке (иконка hamburger)
- При открытии — **полноэкранный overlay** (100vw × 100vh)

### 5.2 Полноэкранное меню
- Фон: белый или светло-серый
- Верхняя зона: логотип + кнопка закрытия
- Контент: вертикальный скролл
  - Группы как секции с заголовком
  - Ссылки — список с иконками
  - Ссылка с subsections: при тапе раскрывается список под ней (аккордеон)

### 5.3 Раскрытие уровней
- Только аккордеон — никаких hover, только tap
- Анимация: `max-height` или `grid-template-rows: 0fr` → `1fr` (300ms ease)
- Один уровень открыт за раз (опционально: можно разрешить несколько)

### 5.4 Закрытие
- Кнопка ✕
- Кнопка «Назад» в нативной шапке (если есть)
- Свайп вниз (опционально)

## 6. Позиционирование подменю (детализация)

### 6.1 Dropdown (desktop)
```
Триггер: [Интеграции ▼]
         ↓
┌─────────────────────────┐
│ Маркетплейс             │
│ ERP, PLM, POS           │
│ Webhooks & API          │
│ SSO                     │
└─────────────────────────┘
```

- `position: fixed`
- Левый верхний угол: `(triggerRect.left, triggerRect.bottom + 4)`
- Проверки:
  - Если `left + width > innerWidth` → `left = innerWidth - width - 16`
  - Если `top + height > innerHeight` → открывать **вверх** (`bottom = innerHeight - triggerRect.top + 4`)
- Тень: `shadow-xl`, `border-radius: 12px`
- `min-width: 200px`, `max-width: 280px`

### 6.2 Боковая панель (tablet)
- Панель справа, контент — вертикальный список
- Подменю — отступ слева 16px, визуальная вложенность (линия или фон)

### 6.3 Полноэкранное (mobile)
- Подменю — тот же список, с отступом 24px слева
- Иконка ChevronRight поворачивается на 90° при раскрытии

## 7. Анимации

| Действие | Desktop | Tablet | Mobile |
|----------|---------|--------|--------|
| Открытие overlay | fade 200ms | slide from right 300ms | fade 250ms |
| Открытие dropdown | fade + translateY(-4) 150ms | — | — |
| Раскрытие аккордеона | — | max-height 300ms ease | max-height 300ms ease |
| Закрытие | fade 150ms | slide 250ms | fade 200ms |

---

## 8. Контроль overflow

### 8.1 Горизонтальный скролл (группы, ссылки)
- Контейнер: `overflow-x: auto`, `overflow-y: hidden`
- `scrollbar-hide` для скрытия скроллбара (визуально)
- `-webkit-overflow-scrolling: touch` для плавного скролла на iOS
- `min-width: 0` на flex-детях, чтобы не раздувать родителя

### 8.2 Вертикальный скролл (dropdown, панель)
- `max-height: calc(100vh - 120px)` для dropdown
- `overflow-y: auto` для длинных списков
- Паддинг снизу, чтобы последний элемент не прилипал к краю

### 8.3 Текст
- `white-space: nowrap` для пунктов меню в desktop
- На mobile — `white-space: normal`, перенос при необходимости
- `text-overflow: ellipsis` при жёстком `max-width`

## 9. Z-index слои

```
50  — Backdrop (затемнение)
51  — Dropdown / Боковая панель
52  — Кнопка закрытия панели (если нужна поверх)
100 — Tooltip (если есть)
```

---

## 10. Горячие клавиши (desktop)

- `Escape` — закрыть открытое меню/dropdown
- `Tab` — навигация по фокусу внутри открытого меню
- При фокусе на пункте с subsections — `Enter` или `→` открывает подменю

---

## 11. Рекомендуемая реализация

1. **Единый компонент** `BrandNav` с пропсами `variant: 'desktop' | 'tablet' | 'mobile'` (или `useMediaQuery` внутри)
2. **Desktop:** текущая двухстрочная навигация + `DropdownMenu` (Radix) для subsections с кастомным позиционированием
3. **Tablet:** `Sheet` (Radix) или кастомная панель справа, контент — аккордеон
4. **Mobile:** `Sheet` на весь экран или отдельный fullscreen-компонент с аккордеоном
5. **Breakpoints:** `md: 768px`, `lg: 1024px` — переключение вариантов по `useMediaQuery` или CSS container queries

---

## 12. Чеклист перед релизом

- [ ] Dropdown не выходит за правый край экрана
- [ ] Dropdown не выходит за нижний край (или переворачивается вверх)
- [ ] На tablet панель 320px не обрезается на узких экранах (min 280px)
- [ ] На mobile контент меню прокручивается при длинном списке
- [ ] Backdrop перекрывает весь экран, клик закрывает меню
- [ ] Escape закрывает любое открытое меню
- [ ] Нет горизонтального overflow на странице при открытом меню
- [ ] Touch targets ≥ 44px на mobile
