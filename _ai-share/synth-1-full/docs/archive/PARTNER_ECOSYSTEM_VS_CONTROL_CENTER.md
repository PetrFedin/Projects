# Партнёрская экосистема vs Центр управления: анализ дублирования и полноты

## Источники

- **Центр управления:** `/brand/control-center` — стратегический хаб, live-виджеты, сетка модулей (MODULE_HUBS), быстрые действия.
- **Партнёрская экосистема:** секция на странице `/brand/organization` — KPI, связь с процессами, цепочки (поставки / продажи / интеграции), 6 блоков (контракты, финансы, качество, логистика, задачи, аналитика).

---

## 1. Дублирование информации

Один и тот же **раздел** или **действие** встречается в обоих местах, но с разным контекстом:

| Тема | В Центре управления | В Партнёрской экосистеме | Оценка |
|------|----------------------|---------------------------|--------|
| **B2B заказы** | Live-виджет «ORD-4521 требует подтверждения», Quick Action «Создать заказ B2B» | «Связь с процессами» (Заказы B2B), карточки Магазины/Дистрибуторы → b2b-orders | Не дубль по смыслу: ЦУ — общий алерт/действие, Партнёрская — контекст по партнёрам. Оставить оба. |
| **Документы** | Quick Action «Документы» | «Связь с процессами» + блок «Контракты и документы» | Частичный дубль. В Партнёрской — документы *с партнёрами*. Имеет смысл в обоих местах; в Партнёрской явно подписывать «документы с партнёрами». |
| **Задачи** | Live-виджет «3 задачи с дедлайном сегодня» → calendar?layers=tasks | «Связь с процессами» (Задачи) + блок «Задачи и действия» | Частичный дубль. В Партнёрской — задачи *по партнёрам*. Оставить оба; в Партнёрской — формулировка «задачи по партнёрам». |
| **Финансы** | Live-виджет «P&L обновлён» → /brand/finance | Блок «Финансы и расчёты» (дебиторка по партнёрам) | Разный угол: ЦУ — общий пульс, Партнёрская — финансы *по партнёрам*. Дублирования по смыслу нет. |
| **Возвраты** | MODULE_HUBS: «Возвраты» → /brand/returns-claims | «Связь с процессами» + блок «Качество и рекламации» | Одна страница, разный контекст (ЦУ — модуль, Партнёрская — возвраты *от партнёров*). Оставить оба. |
| **Production** | Hero, live-виджет «2 сэмпла на проверке», badges | Карточка «Производства» (тип партнёра — фабрики) | Разный уровень: ЦУ = раздел Production, Партнёрская = тип партнёра. Не дубль. |
| **Логистика** | MODULE_HUBS: «Логистика» → /brand/logistics | Блок «Логистика и отгрузки» | Одна тема: в ЦУ — общий модуль, в Партнёрской — логистика в контексте партнёров. Допустимо; в Партнёрской — «логистика по партнёрам/каналам». |

**Рекомендация по дублированию:** не убирать блоки, а везде, где ссылка общая (документы, задачи, возвраты, логистика), явно указывать контекст: «по партнёрам», «с партнёрами», «от партнёров».

## 2. Что есть в Центре управления, но НЕТ в Партнёрской экосистеме

| Элемент ЦУ | Нужно ли добавлять в Партнёрскую? |
|------------|-----------------------------------|
| **Операционный пульс / Dashboard** | Нет. Общий пульс бренда, не партнёрский срез. |
| **Аналитика 360, Клиентский интеллект** | Нет. Стратегия и клиенты, не партнёры. В Партнёрской уже есть «Аналитика по партнёрам» — другой срез. |
| **Health Index, Token Economy** | Нет. Специфика ЦУ, не про партнёров. |
| **Quick Actions: Шоурум, Live, События, Команда, Экспорт** | Шоурум и Команда — опционально добавить в Партнёрскую как «Где работать с партнёрами» (шоурум для байеров, команда — ответственные). Остальное в Партнёрскую не переносить. |
| **MODULE_HUBS: Russian Layer, Продукты, Контент, Маркетинг, ESG, Коммуникации, HR, Академия, Лояльность, Соцсети** | В Партнёрскую не добавлять — не про партнёров. |
| **Аукционы** (/brand/auctions) | Имеет смысл добавить: закупки у поставщиков. В Партнёрской — ссылка в блок «Поставщики» или отдельная строка «Закупки / Аукционы». |

Итого: в Партнёрскую экосистему из ЦУ стоит добавить только **Аукционы (закупки)** в контекст поставщиков; по желанию — короткие ссылки на **Шоурум** и **Команду** как точки работы с партнёрами.

## 3. Что есть в Партнёрской, но слабо представлено в ЦУ

- **Сводка по типам партнёров** (производства, поставщики, магазины, дистрибуторы, интеграции) с цифрами и трендами — в ЦУ такой сводки нет, только отдельные модули B2B, Поставщики, Дистрибьюторы.
- **Роли:** цепочка поставок / каналы продаж / связь и интеграции — только в Партнёрской.
- **Блоки:** контракты, финансы по партнёрам, качество и рекламации, логистика, задачи по партнёрам, аналитика по партнёрам — в ЦУ нет единого среза «всё про партнёров».

Это не дублирование, а усиление: Партнёрская экосистема даёт именно партнёрский срез, которого в ЦУ нет.

## 4. Всё ли подходит под раздел «Партнёрская экосистема»

| Блок в Партнёрской | Подходит? |
|--------------------|-----------|
| Цепочка поставок (производства, поставщики) | Да. Партнёры по поставкам. |
| Каналы продаж (магазины, дистрибуторы) | Да. Партнёры по сбыту. |
| Связь и интеграции | Да. Связь с партнёрами (1С, ЭДО, маркетплейсы). |
| Связь с процессами (заказы B2B, документы, задачи, возвраты, отгрузки) | Да. Все процессы, где участвуют партнёры. |
| Контракты и документы | Да. Документы с партнёрами. |
| Финансы и расчёты | Да. Расчёты с партнёрами. |
| Качество и рекламации | Да. Возвраты и претензии от партнёров. |
| Логистика и отгрузки | Да. Отгрузки партнёрам и по каналам. |
| Задачи и действия | Да. Задачи по партнёрам. |
| Аналитика по партнёрам | Да. Топ партнёры, план/факт по каналам. |

Все перечисленные блоки уместны в разделе «Партнёрская экосистема».

## 5. Рекомендации

1. **Дублирование:** Не удалять блоки. Где ссылка общая с ЦУ (документы, задачи, возвраты, логистика) — в Партнёрской везде подчёркивать контекст: «по партнёрам», «с партнёрами», «от партнёров».
2. **Добавить в Партнёрскую из ЦУ:**
   - **Аукционы (закупки)** — в контекст поставщиков (ссылка в карточке «Поставщики» или отдельная строка «Закупки»).
   - По желанию: короткие ссылки **Шоурум** (работа с байерами) и **Команда** (ответственные за партнёров) в блок «Связь с процессами» или отдельной строкой.
3. **Не переносить из ЦУ в Партнёрскую:** Операционный пульс, Аналитика 360, Клиентский интеллект, Health Index, Token Economy, общие Quick Actions (Live, События, Экспорт), модули не про партнёров.
4. **Состав Партнёрской экосистемы:** Текущий набор блоков под раздел подходит; лишнего нет.
