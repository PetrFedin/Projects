## 📊 КОНЦЕПЦИЯ МАТРИЦЫ

### **Что такое Матричная Структура**:

Вместо традиционного набора виджетов, B2B рабочее место организовано как **интерактивная матрица**:

```
┌────────────────────────────────────────────────┐
│         РОЛЬ: Ретейлер / Байер / Бренд         │
│         ОРГАНИЗАЦИЯ: [Switcher]                │
└────────────────────────────────────────────────┘
              ↓
┌────────────────────────────────────────────────┐
│  ТАБЫ (Категории функций):                     │
│  • Аналитика и Инсайты                         │
│  • Операции и Планирование                     │
│  • Совместная Работа                           │
│  • Финансы и Оплата                            │
└────────────────────────────────────────────────┘
              ↓
┌────────────────────────────────────────────────┐
│  КАРТОЧКИ ФУНКЦИЙ (фильтр по роли):            │
│  [Рыночная Аналитика] [AI Рекомендации] ...    │
│                                                │
│  Каждая карточка:                              │
│  • Иконка + приоритет (критично/важно/средне)  │
│  • Описание функции                            │
│  • Метрики в реальном времени                  │
│  • Теги ролей кто имеет доступ                 │
│  • Кнопка "Открыть"                            │
└────────────────────────────────────────────────┘
              ↓
┌────────────────────────────────────────────────┐
│  ЛЕГЕНДА РОЛЕЙ:                                │
│  Матрица прав доступа для всех ролей           │
└────────────────────────────────────────────────┘
```

### **Метрики**:

Каждая карточка может показывать до 2-3 метрик:
- **Label**: Название метрики (uppercase, font-bold)
- **Value**: Значение (tabular-nums, font-black)
- **Trend**: ↑ Up (green) | ↓ Down (red) | → Neutral (gray)

Примеры:
```
Trending Items:  12 ↑
Price Position:  +8% ↑
Match Score:     92% ↑
Low Stock:       12 ↓
```

## 🔄 СИНХРОНИЗАЦИЯ С ПРОФИЛЯМИ

### **Источник данных**:

```typescript
// 1. User Context Hook
const userContext = useUserContext();
const { isRetailer, isBrand, isBuyer, currentOrg, user } = userContext;

// 2. Определение Primary Role
const primaryRole = useMemo(() => {
  if (isRetailer) return 'retailer';
  if (isBrand) return 'brand';
  if (isBuyer) return 'buyer';
  return 'buyer'; // default
}, [isRetailer, isBrand, isBuyer]);

// 3. Фильтрация по ролям
const allowedItems = ROLE_PERMISSIONS[primaryRole] || [];
const filteredItems = WORKSPACE_ITEMS
  .filter(item => item.category === activeTab)
  .filter(item => allowedItems.includes(item.id));
```

### **Связь с Organization**:

- **Название организации**: `currentOrg?.name`
- **Логотип**: `currentOrg?.logo`
- **Кредитный лимит**: `currentOrg?.creditLine`
- **Loyalty tier**: `currentOrg?.loyaltyTier`

### **Связь с User Profile**:

- **Имя**: `user?.displayName`
- **Email**: `user?.email`
- **Роли**: `user?.roles[]`
- **Permissions**: `user?.permissions[]`

## 🚀 ПРЕИМУЩЕСТВА МАТРИЧНОЙ СТРУКТУРЫ

### **1. Персонализация на уровне роли**:
- Каждый пользователь видит только свои функции
- Нет cognitive overload
- Ясная иерархия приоритетов

### **2. Масштабируемость**:
- Легко добавить новую роль → просто описать permissions
- Легко добавить новую функцию → просто описать item
- Легко изменить структуру → редактируем JSON data

### **3. Transparency (Прозрачность)**:
- Легенда ролей показывает ВСЕ роли и их доступы
- Пользователь понимает свои права
- Понятно кому писать за расширением доступа

### **4. Визуальная ясность**:
- Color coding по приоритету
- Иконки для быстрого распознавания
- Badges для статусов
- Метрики в реальном времени

### **5. No Duplication**:
- Один source of truth (WORKSPACE_ITEMS)
- Нет дублирования кода виджетов
- Единая точка обновления
