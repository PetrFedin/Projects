# Стандарт оформления кабинетов (эталон: Академия)

Одна страница: как использовать **Академию** как визуальный и структурный эталон для остальных кабинетов, не дублируя полностью `STYLE_GUIDE.md` и `src/design/UI_RULES.md`.

## 1. Связь с существующими источниками правды

| Документ / код | Роль |
|----------------|------|
| `STYLE_GUIDE.md` | Глобальная типографика и Tailwind |
| `src/design/UI_RULES.md`, `src/design/tokens.json` | Семантические токены |
| `src/lib/ui/cabinet-surface.ts`, `.cursor/rules/cabinet-ui-consistency.mdc` | Единый «лист» кабинета, `cabinetSurface` |
| `src/lib/ui/nuorder-desk-shell.ts` → `nuOrderDeskShell` | Плотный **desk**-канвас (границы `#c5ccd6`, ссылки `#0b63ce`, мелкий текст) |
| `src/components/design-system/` | Примитивы: `PageContainer`, `RegistryPageHeader`, карточки метрик и т.д. |
| `src/design/JOOR_ORACLE_ENTERPRISE_UI.md` | Плотные B2B-экраны (не смешивать с маркетинговым hero без осознанного выбора) |

**Правило:** новые экраны сначала выбирают **тип страницы** (ниже), затем токены/примитивы из таблицы выше — не новые произвольные hex, если есть семантический аналог.

## 2. Токены «как в академии» (маппинг на семантику)

Академия в client-контуре часто сидит на **`nuOrderDeskShell.canvas`** или повторяет те же значения inline.

| Назначение | Типичные значения в эталоне | Куда маппить в системе |
|------------|----------------------------|-------------------------|
| Фон рабочей области | `#e6e9ef`, серый холодный | `bg-muted` / surface кабинета / `cabinetSurface` |
| Карточка / панель | `bg-white`, бордер `#c5ccd6`, `rounded-sm`–`rounded-2xl` по уровню | `border-border` / `bg-card`, радиус по слою |
| Первичное действие / ссылка | `#0b63ce`, hover темнее | `primary` / `text-accent-primary` / ссылки реестра |
| Подписи, вторичный текст | `#6b7788`, `#5b6675` | `text-muted-foreground` / `text-text-secondary` |
| Заголовок страницы объекта | `#1a2433`, 18–20px semibold | `text-foreground` / headline в shell |
| Секционный лейбл | 10–11px, uppercase, wide tracking | Единый класс/компонент «SectionEyebrow» (см. §4) |

**Не хардкодить** в новых фичах: дублировать `#0b63ce` десятком раз — лучше класс-обёртка или токен из `tokens.json`, если он уже есть для primary.

## 3. Три типа страниц (обязательный выбор перед вёрсткой)

### A. Hub (витрина, обзор)

- **Признаки:** hero или крупный блок вверху, блок метрик, затем вкладки/каталог.
- **Эталон в репозитории:** `src/app/academy/page.tsx` — секция с градиентом, `#academy-tabs`, контейнер `max-w-5xl`.
- **Радиусы:** наружные `rounded-2xl`, внутри списков можно плотнее.
- **Когда использовать:** главные `/academy`, хабы разделов, лендинги внутри кабинета.

### B. Object detail (карточка сущности)

- **Признаки:** хлебные крошки, одна основная колонка, блоки «секциями», без hero.
- **Эталоны:**  
  - `src/app/academy/course/[id]/page.tsx` — обложка, мета-строка, программа, отзывы, CTA.  
  - `src/app/academy/path/[id]/page.tsx` — прогресс, табы, модули.
- **Оболочка:** чаще `nuOrderDeskShell.canvas` + внутренний `rounded-sm border` у белой карточки.
- **Когда использовать:** курс, программа, статья, заказ, карточка справочника.

### C. Dense tool (плотный инструмент)

- **Признаки:** мало «воздуха», фильтры, таблицы, статусы; hero нет.
- **Эталон:** не академия, а `OPERATIONAL_SCREEN_SPEC.md`, `DataTableContainer`, матрица B2B — но **те же токены** границ и текста, что и у desk.
- **Когда использовать:** списки заказов, инвентарь, админские таблицы.

**Ошибка:** натянуть hub-hero на экран типа C — визуальный шум и потеря плотности.

## 4. Два примитива секции (стандартизировать, а не копипастить `div`)

Рекомендуемая декомпозиция для выравнивания с академией без дублирования разметки:

| Примитив | Состав | Эталон в Академии |
|----------|--------|-------------------|
| **Section shell** | Eyebrow (uppercase label) + опционально `Separator` / `border-t` + контент в `space-y-*` | Блоки «Программа», «Отзывы», «Материалы» на странице курса |
| **Context strip** | Одна строка или компактная карточка над основным блоком: контекст пользователя (напр. «Моё обучение», чаты) | `AcademyClientLearningBar` на `/academy`, курсе, траектории |

На стороне **design-system** уже есть `SectionContainer`, `PageHeader` и др. — при новом коде **сначала** смотреть, хватает ли их; если нет — расширить примитив, а не плодить локальные копии с другими отступами.

Имена для ревью PR:

- «Это Object detail — почему hub-hero?»
- «Лейбл секции совпадает с эталоном Section shell (размер/tracking)?»
- «Плотность как dense tool — обёртка `DataTableContainer` / operational?»

## 5. Что академия **не** покрывает (договориться отдельно)

- **Брендовый кабинет** (`CabinetPageContent`, `RegistryPageHeader`): другая воздушность — стандарт = те же токены, но **другой тип страницы** (часто hub + реестр, не nuOrder canvas).
- **Таблицы с десятками колонок:** эталон operational / JOOR Oracle, не карточка курса.
- **Маркетинговые публичные страницы** вне кабинета — не смешивать с `nuOrderDeskShell` без отдельного дизайн-решения.

## 6. Эталонные файлы (быстрая навигация для агентов и ревью)

| Тип | Путь |
|-----|------|
| Hub | `src/app/academy/page.tsx` |
| Object detail (курс) | `src/app/academy/course/[id]/page.tsx` |
| Object detail (траектория) | `src/app/academy/path/[id]/page.tsx` |
| Полоса контекста + чаты | `src/components/academy/academy-client-learning-bar.tsx` |
| Desk-оболочка | `src/lib/ui/nuorder-desk-shell.ts` |
| Платформа бренда (другая плотность) | `src/app/brand/academy/platform/page.tsx` |

## 7. Чеклист для нового экрана в кабинете

1. Выбран тип **A / B / C** (§3).  
2. Границы и фоны из §2 или из design tokens — без новых серых «на глаз».  
3. Секции оформлены как **Section shell** или существующий примитив из `design-system`.  
4. Маршруты только через `ROUTES.*` (`src/lib/routes.ts`).  
5. Адаптив: см. `responsive-design.mdc`; плотные экраны — проверка 1024 и 375.
