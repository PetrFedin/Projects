# Минимум полей заказа / отгрузки под РФ (v1)

**Статус:** **черновик продукта** — юридический sign-off не выполнялся; обязательное согласование с юристом/комплаенсом — **до** продакшен-интеграции ЭДО и юридически значимых отгрузок.

**Допущение для старта этапа 1:** зафиксировано в [`PHASE-0-CLOSURE-CHECKLIST.md`](./PHASE-0-CLOSURE-CHECKLIST.md) (§A4). Риск **R-06** снимается для этапа 1 только вместе с этим допущением (не заменяет найм юриста к продакшену).

**Связь:** [ADR-0003](./ADR/ADR-0003-edo-rf-fields-vs-integration.md)

Цель v1 — не заменить бухгалтерию, а **не потерять** обязательные для переговоров и последующей интеграции ЭДО атрибуты.

## Заказ (контрагент / сделка)

| Поле | Назначение | Примечание |
|------|------------|------------|
| Покупатель (ИНН/КПП или аналог для юрлица) | Идентификация в документах | Ручной ввод допустим в v1 |
| Продавец (бренд) — реквизиты | Сторона УПД | Справочник бренда |
| Валюта | Договорная | **Только RUB** в продукте v1 |
| НДС: ставка / признак | Счёт-фактура / УПД | Перечень ставок — по политике бренда |
| База: сумма без НДС / с НДС | Согласование с заказом | Связь со строками |

## Отгрузка / закрытие

| Поле | Назначение |
|------|------------|
| Номер и дата первичного документа (УПД) | Учёт отгрузки |
| Статус ЭДО (агрегат: не отправлен / в пути / принят) | Процесс без обязательной интеграции в v1 |
| Связь с `wholesaleOrderId` | Сквозная трассировка |

## Маркировка (при наличии КИ в категории)

| Поле | Назначение |
|------|------------|
| Идентификатор КИ / статус вводимости (агрегат) | Контроль перед отгрузкой |
| Примечание по партии | Ручной контур до интеграции ГИС МТ |

**Дальнейшая детализация** (обязательные поля по отраслевым регламентам) — ответственность владельца комплаенса; обновления через ADR или приложение к этапу 2.
