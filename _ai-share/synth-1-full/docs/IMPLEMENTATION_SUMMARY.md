## 🔄 Полный Flow работы

### Регистрация и вход
1. Пользователь заходит на `/login`
2. Регистрируется или входит (тестовый: `test@example.com` / `password123`)
3. Данные сохраняются в localStorage
4. AuthProvider обновляет состояние

### Покупка товара
1. Пользователь просматривает каталог
2. Добавляет товар в корзину → сохраняется в localStorage
3. Переходит в корзину → данные загружаются из localStorage
4. Нажимает "Оформить заказ" → переход на `/checkout`

### Оформление заказа
1. Заполняет форму доставки
2. Нажимает "Оплатить"
3. Создается payment intent (mock)
4. Платеж подтверждается (mock - всегда успешен)
5. Создается заказ в localStorage
6. Корзина очищается
7. Редирект на `/orders/{orderId}`

### Просмотр заказов
1. Пользователь заходит на `/orders`
2. Видит список всех своих заказов
3. Может открыть детали заказа
4. Видит статус, товары, адрес доставки

### AI Stylist
1. Пользователь открывает AI Stylist (кнопка внизу справа)
2. Вводит запрос (например, "образ для свадьбы")
3. AI генерирует изображение образа
4. Система ищет подходящие товары в каталоге
5. Показывает рекомендации с возможностью добавить в корзину

## 🔧 Технические детали

### Хранение данных
Все данные хранятся в localStorage:
- `syntha_auth_user` - текущий пользователь
- `syntha_users` - база пользователей
- `syntha_cart_{userId}` - корзина
- `syntha_wishlist_{userId}` - wishlist
- `syntha_wishlist_collections_{userId}` - коллекции wishlist
- `syntha_orders_{userId}` - заказы
- `syntha_payment_intents` - платежи

### Realtime обновления
Симулируются через:
- Listeners в репозиториях
- Автоматический вызов callback при изменении данных
- Синхронизация через useEffect в компонентах

### Замена на Firebase
Для замены на Firebase нужно:
1. Создать `src/lib/repositories/firebase/` с реализациями
2. Обновить `src/lib/repositories/index.ts` - заменить экспорты
3. Все компоненты продолжат работать без изменений!
