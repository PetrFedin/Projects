## Запуск

Версия Node **20.x–23.x** (см. **`.nvmrc`**, **`engines`** в **`package.json`**). Скрипты **`test:e2e*`** сначала вызывают **`scripts/ensure-supported-node.mjs`** — на Node 24+ команда завершится с подсказкой **`nvm use`**, до старта Playwright/dev-сервера.

```bash
npm run test:e2e
```

Playwright поднимает **`npm run dev:e2e`** (Next на **`http://127.0.0.1:3123`**, не занимает :3000). Другой URL: **`PLAYWRIGHT_BASE_URL`**. В CI: **`CI=1`** — всегда новый сервер на этом порту.

В корне монорепозитория **`/.github/workflows/synth-1-full-ci.yml`** для E2E задаётся **`NEXT_PUBLIC_USE_B2C_SHOP_API=true`**, чтобы сценарии shop API ходили в `/api/shop/*`. На каждом PR по умолчанию гоняются **`test:e2e:light`**, **`test:e2e:cabinet-hubs`**, а **`test:e2e:shop-retail`** — только если в PR менялись пути ритейл-кабинета (см. job **`changes`** / **`paths-filter`** в workflow). Полный пакет **`test:e2e:api`** + verification — job **`ci-heavy`** (**`push`** в **`main`**, метка PR **`ci-heavy`**, еженедельный **cron** или **`workflow_dispatch`**); визуальные тесты — **`ci-visual`** или **`workflow_dispatch`**. Standalone **`_ai-share/synth-1-full/.github/workflows/ci.yml`** — то же условие для **`shop-retail`**.

## Расширение a11y (axe)

Для проверки по правилам WCAG (контраст, метки форм, ARIA) можно добавить [@axe-core/playwright](https://www.npmjs.com/package/@axe-core/playwright):

```bash
npm i -D @axe-core/playwright
```

В тесте:

```ts
import { AxeBuilder } from '@axe-core/playwright';

const results = await new AxeBuilder({ page }).withTags(['wcag2a', 'wcag2aa']).analyze();
expect(results.violations).toEqual([]);
```
