## Приоритет 1 — быстрые победы

### 1. Недавняя активность: даты относительно «сегодня»
В `page-data.ts` у `RECENT_ACTIVITIES` даты зафиксированы (2026-03-09 … 2026-03-14). При выборе «7 дней» от текущей даты события часто не попадают в диапазон и список пустой.  
**Сделать:** генерировать `dateStr` относительно `new Date()` (например, сегодня −1…−5 дней), чтобы при смене периода в ленте всегда были события.

### 2. Шапка: «24 участника» и «8 онлайн» из API
Сейчас значения статичные. Когда появится эндпоинт команды/присутствия (например в dashboard или `GET /team/summary`) — подставлять реальные числа в бейджи «24 участника» и «8 онлайн».

### 3. Ошибки и пустые состояния
- При ошибке загрузки profile/dashboard показывать короткое сообщение («Не удалось загрузить данные») и кнопку «Повторить» (refetch).
- Если после загрузки profile пустой — не ломать шапку, оставить fallback «Syntha HQ».

## Приоритет 2 — данные с бэкенда

### 4. Алерты «Требует внимания» из API
`use-attention-alerts.ts` сейчас с моковыми `INITIAL`.  
**Варианты:**  
- Эндпоинт `GET /api/v1/organization/alerts` (или включить алерты в ответ `organization/health` / dashboard).  
- Фронт инициализирует состояние из ответа; dismiss по желанию сохранять в state или через API.

### 5. Brand/dashboard — реальные агрегаты
В `app/api/v1/endpoints/brand.py` стоит TODO: подгружать profile/dashboard из БД по `organization_id`/`brand_id`, агрегировать retailersCount, openB2bOrders, markingSyncStatus и т.д. из репозиториев (orders, retailers, compliance).

### 6. Карточки модулей: цифры из API
У `NAVIGATION_CARDS` статичные `stats` (например «92%», «24», «2»). Имеет смысл эндпоинт вида `GET /api/v1/brand/module-stats` или расширить ответ dashboard и на фронте подставлять эти значения в карточки.

### 7. Партнёрская экосистема: данные по периоду
`PARTNER_COUNTS`, `PARTNER_GROWTH_BY_PERIOD` и др. пока из `page-data`. Дальше — запросы с параметром периода (7d/30d) и подстановка в блоки «Партнёры по типам», «Рост за период», «Связь с процессами».

## Приоритет 3 — полировка

### 8. Доступность (a11y)
- Проверить aria-label у всех интерактивных элементов в блоках «Требует внимания» и «Модули».
- Фокус и навигация с клавиатуры (кнопки периода, календарь, карточки).

### 9. Скелетоны при загрузке
Там, где данные приходят с API (шапка при первом загрузке профиля, партнёрская экосистема при появлении API), добавить скелетоны вместо резкого появления контента.

### 10. Organization health с бэкенда
Сейчас `/organization/health` возвращает `[]`, фронт сам считает метрики из profile/dashboard/integrations. Опционально: бэкенд считает агрегированные метрики и возвращает их в `data`, фронт использует при наличии.
