# Чеклист PR: оформление кабинетов и профилей

**Назначение:** фиксировать единый стандарт при ревью PR, которые меняют UI **личных кабинетов** (маршруты под `/admin`, `/brand`, `/shop`, `/factory`, `/distributor`, `/client`, `/u` и вложенные разделы).

**Источник правды:** `FIGMA_SPEC_PACK_RU.md` (компоненты §2, токены §3, локализация §4, приёмка §5–6). **Расширенный канон кабинетов и профилей (код + пробелы + роли):** `src/design/CABINET_PROFILES_STYLING_RULES_RU.md`. Этот файл — **сжатый чеклист для описания PR**; детали и обоснования — там и в каноне профилей.

---

## Как использовать

1. В описании PR вставьте блок чеклиста ниже и отметьте пункты (`[x]` / `[ ]`).
2. Если пункт **не выполнен** намеренно — после чеклиста добавьте строку **«Исключения»** с краткой причиной (ссылка на тикет, временный демо-слой, ограничение сроков).
3. Ревьюер проверяет соответствие; при споре — §2–6 основного spec pack.

---

## Блок для вставки в описание PR

```markdown
### Кабинеты: оформление (FIGMA spec / токены)

- [ ] **Оболочка:** фон контентной области согласован с `bg-bg-surface` (или обосновано); сайдбар/шапка роли не ломают общий паттерн.
- [ ] **Шапка раздела:** крошки + заголовок + действия в том же ритме, что у соседних разделов этой роли; для **корневого layout кабинета** — `CabinetHubMain` + `CabinetHubTitleRow` + `CabinetHubSectionBar` (`src/components/layout/cabinet-hub-chrome.tsx`). Для страниц внутри — `RegistryPageShell` / `RegistryPageHeader` или согласованный аналог.
- [ ] **Токены:** семантические классы (`bg-bg-surface`, `bg-bg-surface2`, `border-border-subtle` / `border-border-default`, `text-text-primary` / `secondary` / `muted`, `text-accent-primary`); без произвольных hex/rgb в разметке страницы.
- [ ] **Вкладки / вторичная навигация:** при наличии Tabs — `cabinetSurface.tabsList` + `cabinetSurface.tabsTrigger` (или `groupTabList` / `groupTabButton*`), без одноразовых стилей.
- [ ] **Реестры / таблицы:** `registryFeedLayout` + плотность `dense` где уместно; строки/панели фильтров на `bg-bg-surface2` и `border-border-subtle`.
- [ ] **Состояния:** заданы пустое, загрузка и ошибка (не «голый» текст без отступов/типографики).
- [ ] **RU + аббревиатуры:** пользовательский текст на русском; аббревиатуры из списка — с `AcronymWithTooltip` при первом появлении на экране (см. §4.2 основного spec pack).
- [ ] **Исключения:** нет / перечислены ниже: _…_
```

## Краткие напоминания по коду (§2–3)

- Поверхности: страница — `bg-bg-surface`, вторичные блоки — `bg-bg-surface2`.
- Акценты: `bg-accent-primary`, `text-accent-primary`, мягкая подложка `bg-accent-primary/10`.
- Единая оболочка контента: предпочтительно `RegistryPageShell` + `RegistryPageHeader` там, где уже принято в разделе.
- Вкладки кабинета: импорт `cabinetSurface` из `@/lib/ui/cabinet-surface`.

При расхождении с Figma приоритет у **токенов и паттернов** из spec pack; расхождение с макетом фиксируется в тикете дизайна или в «Исключениях» PR.
