# План UI-спринтов

## Контур и ограничения

- План покрывает UX главной страницы и глобальной навигации.
- Спринт 1 — без архитектурных переделок и миграций маршрутов.
- Спринт 2 — допускает role-aware главную, персонализацию и умный поиск.
- Канон ролевых B2B-возможностей: `B2B_CATALOG_ROLE_CAPABILITY_MATRIX_RU.md`.
- Следующие секции являются визуальным эталоном и не редизайнятся:
  - `PARTNERS_b2c`
  - `SHOWCASE_b2c`
  - `LABORATORY_B2C`
  - `MEDIA_ECOSYSTEM_b2c`

## Спринт 1 — быстрые UX-победы (без ломки архитектуры)

### A. Навигация и информационная архитектура

- [ ] Зафиксировать B2C верхнее меню: `АССОРТИМЕНТ`, `БРЕНДЫ`, `LIVE`, `ЛОЯЛЬНОСТЬ`.
- [ ] B2B верхнее меню оставить платформенным входом, без дублей кабинетов.
- [ ] Убрать дублирующиеся по смыслу пункты.
- [ ] Синхронизировать desktop/mobile подписи пунктов.

Критерии приемки:

- Нет дублей в верхнем уровне навигации.
- Переключатель B2C/B2B меняет только глобальные entry points.
- Нет runtime-предупреждений/ошибок по ссылкам меню.

### B. Иерархия первого экрана

- [ ] Сохранить порядок: рекламный баннер -> селектор разделов -> ключевой контент.
- [ ] Нормализовать вертикальный ритм отступов в первых двух экранах.
- [ ] Уменьшить количество конкурирующих анимационных акцентов.

Критерии приемки:

- Основной сценарий считывается за 3 секунды.
- Нет визуальных скачков между шапкой, селектором и первым блоком.

### C. Визуальная консистентность и полировка

- [ ] Привести иерархию кнопок к единому паттерну (primary/secondary/ghost).
- [ ] Сохранить акцент `LIVE`, но без артефактов рамки/прозрачности.
- [ ] Убрать декоративные эффекты, ухудшающие читаемость.

Критерии приемки:

- Состояния active/inactive читаемы и стабильны.
- Нет визуальных артефактов в header-анимациях.

### D. Снижение шума в связанных ссылках

- [ ] Перевести related links в компактный режим (топ + раскрытие).
- [ ] Отфильтровывать невалидные ссылки до рендера.

Критерии приемки:

- Нет ошибок `href` на runtime.
- Блок связанных ссылок не перетягивает фокус с основного сценария.

## Спринт 2 — role-aware главная + персонализация + умный поиск

### E. Role-aware главная

- [ ] Добавить role-варианты первого экрана для `client`, `shop`, `brand`, `factory`, `supplier`, `admin`.
- [ ] Сохранить общий визуальный каркас, меняя только контент.

Критерии приемки:

- Для каждой роли видны релевантные первые действия.
- Визуальный стиль остается согласованным с эталонными секциями.

### F. Персонализация

- [ ] Добавить `Продолжить с места`.
- [ ] Добавить `Закрепленные модули`.
- [ ] Добавить `Недавние действия`.

Критерии приемки:

- Возврат к последнему важному сценарию — в один клик.
- Закрепленные модули сохраняются между сессиями.

### G. Умный глобальный поиск

- [ ] Единая точка поиска по брендам, модулям, заказам и SKU.
- [ ] Быстрые действия прямо из результатов.
- [ ] Описанные состояния: loading/empty/no-access.

Критерии приемки:

- Результаты смешанных типов с явными type-label.
- Клавиатурный сценарий open -> navigate -> submit работает стабильно.

## QA-чеклист

- [ ] Нет React key warnings на главной и в сайдбарах.
- [ ] Нет runtime ошибок `Link href=undefined`.
- [ ] Переключение B2C/B2B стабильно на desktop/mobile.
- [ ] Header и селекторы стабильны после hard refresh.
- [ ] Эталонные секции визуально не изменены.
