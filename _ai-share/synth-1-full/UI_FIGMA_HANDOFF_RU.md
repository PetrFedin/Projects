# UI Figma Handoff (RU)

## Цель

Зафиксировать структуру handoff в Figma для обновлений главной страницы в Спринте 1 и Спринте 2.

## Структура файла в Figma

- `00_Cover`
- `01_Current_State`
- `02_Sprint_1_Quick_Wins`
- `03_Sprint_2_Role_Aware`
- `04_Personalization`
- `05_Smart_Search`
- `99_Locked_References`

## Правила именования

- Фреймы: `HOME/<mode>/<section>/<state>`
  - Пример: `HOME/B2C/HERO/default`
  - Пример: `HOME/B2B/NAV/active-live`
- Компоненты: `CMP/<group>/<name>/<variant>`
  - Пример: `CMP/Nav/TopLink/live`
- Переменные:
  - `color.*`
  - `space.*`
  - `radius.*`
  - `motion.*`

## Locked vs Editable

### Locked (не редизайнить)

- `PARTNERS_b2c`
- `SHOWCASE_b2c`
- `LABORATORY_B2C`
- `MEDIA_ECOSYSTEM_b2c`

Правила:

- Не менять визуальный язык, пропорции карточек и иерархию типографики.
- Допускаются только контентные обновления (текст/медиа), без стилевого дрейфа.

### Editable

- Поведение и подписи верхней навигации.
- Ряды селекторов под баннером (B2C категории и B2B кнопки разделов).
- Иерархия первого экрана, ритм отступов, иерархия CTA.
- Компактный режим для related links.
- Role-aware блоки первого экрана (Спринт 2).
- Виджеты персонализации и UX поиска (Спринт 2).

## Дизайн-чеклист

- [ ] Нет дублирующихся по смыслу пунктов в навигации.
- [ ] Поведение переключателя B2C/B2B прозрачно и предсказуемо.
- [ ] В каждом viewport один доминирующий action path.
- [ ] Анимации не ухудшают читаемость и визуальную стабильность.
- [ ] Описаны состояния: default, hover, active, disabled, loading, empty.

## Артефакты handoff

- Аннотированные фреймы для каждой измененной секции.
- Спеки компонентов (отступы, размеры, состояния).
- Примечания по интеракциям hover/focus/active.
- Сравнение before/after для quick wins Спринта 1.
- Матрица role-вариантов (client/shop/brand/factory/supplier/admin) для Спринта 2.

## Примечания для разработки

- Сохранять существующую структуру маршрутов и границы ролевых кабинетов.
- Не вводить новые визуальные примитивы при наличии подходящих из дизайн-системы.
- Locked-секции считать каноном для всех следующих итераций.
