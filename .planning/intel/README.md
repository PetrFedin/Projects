# `.planning/intel/` — индекс кодовой базы (GSD Intel)

Каталог для **запрашиваемого интеллекта** по репозиторию: сюда попадают сгенерированные индексы после **`/gsd-intel refresh`** (навык **`gsd-intel`**, агент **`gsd-intel-updater`**), если в **`.planning/config.json`** включено **`intel.enabled`**.

## Что делать агенту

1. Перед длинным «как устроен весь репозиторий» — **`/gsd-intel query …`** или **`status`**, а не бесконечный `grep` по корню.
2. Если каталог пустой или устарел — инициировать **`refresh`** (через навык/команду GSD), затем опираться на появившиеся файлы индекса.
3. **Не править вручную** автогенерируемые JSON/артефакты индекса — только через GSD; этот README — исключение (документация для людей и агентов).

## Что делать человеку

- Разово после включения intel: выполнить **`/gsd-intel refresh`** в Cursor (или эквивалент через GSD CLI), дождаться заполнения каталога.
- При смене крупной архитектуры — повторить **`refresh`**.
