# Инвесторское демо: три столба «максимально рабочие» (честно про mock)

**Дата:** 2026-05-11 · **Канон:** `_ai-share/synth-1-full` · **Опора:** `FOCUS_ONE_PAGER.md`, `PLAN_RESTRUCTURE_THREE_PILLARS.md` (§2–6), `e2e/helpers/w2-demo-routes.ts`, `AGENTS.md` (smoke / tech pack / B2B).

## 1. Суть демо (60–90 секунд)

**Elevator:** Fashion OS — один Next.js с кабинетами бренда, ритейла и смежных ролей; мы показываем не «меню на 200 экранов», а **три связанные оси операций моды**: (A) разработка артикула ТЗ → образец, (B) коллекция и оптовый контур до **операционного заказа**, (C) **сообщения и календарь** как сквозной слой согласований.

**Что доказываем за минуту:** зрелый **UI + BFF-контуры** под приёмочными смоками; **единая навигация** между производством, B2B и коммуникациями; **прозрачная подпись** где данные демо/localStorage/seed, где API read-model, где пилот (tech pack) зависит от env.

## 3. Минимальная подготовка окружения

- **Node:** 20–23 (`.nvmrc` во full); `**npm ci`** в `_ai-share/synth-1-full`; `**npm run dev`** → `http://localhost:3000` (или прокси из корня монорепо — см. `AGENTS.md`).
- **Демо-tenant / логин:** использовать существующий демо-bootstrap из канона (`**src/lib/auth/demo-hub-email.ts`**, tenant inventory в матрице `UNIFIED_ECOSYSTEM_VERIFICATION.md` / `INTEGRATION_MAP.md` — **конкретный email: уточнить в коде** для вашей среды).
- **W2 демо-slug:** только `**SS27`**, артикулы `**demo-ss27-01`** / `**demo-ss27-02**` — контракт e2e (`w2-demo-routes.ts`); не подставлять случайные числовые id.
- **Tech pack (только если показываете A3):** скопировать фрагмент из `**env.w2-techpack.example`** в `.env.local`, прогнать `**npm run w2:techpack:preflight`**; иначе шаг A3 вычеркнуть из сценария.
- **Префлайт перед встречей (рекомендация из плана, §6):** из корня `**npm run smoke`**; во full — `**npm run test:e2e:light`**, `**npm run test:e2e:verification**`, `**playwright test e2e/workshop2-smoke.spec.ts**`; API — `**npm run test:e2e:api**` по договорённости.

## 4. Что не показывать в этой сессии

- Весь хвост `**/shop/b2b/***` (margin, отчёты, десятки табов) без одного выбранного сценария.
- **Academy**, **client sewing patterns**, **админские метрики досье** как центр питча.
- `**/brand/planning`** с AI-симуляцией без связки «план ↔ заказ/артикул».
- Organization hub как «живые прод-метрики» без подписи **mock** (см. матрицу в `FUNCTION_SPINE…` / one-pager).
- Python FastAPI без одной фразы «что из этого видно в Next сейчас».

---

## 5. Fallback (план B)

1. **W2 не грузится / пустое досье:** открыть hub + объяснить смок-контракт; при необходимости очистить LS-ключ `**synth.brand.workshop2Phase1Dossier.v1`** и перезагрузить демо-артикул — **уточнить в коде** поведение merge для `demo-ss27-01`.
2. **B2B API 4xx/5xx:** остаться на UI реестра с **пустым состоянием** + коротко показать `**INTEGRATION_MAP`** / один экран интеграций; не импровизировать второй источник данных в UI.
3. **Tech pack недоступен:** одна фраза «пилот с S3/Postgres, сегодня показываем только досье и процесс согласований».
4. **Сообщения/календарь пустые:** empty states + навигация; не обещать доставку сообщений между ролями в реальном времени.

## 6. Скрипт для ведущего (буллеты по экранам)

- **Профиль `/brand/profile`:** «Точка входа бренда в экосистему; дальше три опоры — производство артикула, опт, коммуникации».
- **W2 hub `/brand/production/workshop2`:** «Здесь живёт ТЗ фазы 1: секции, версии, signoff — то, что уходит в фабрику как договорённость, а не PDF в почте».
- **Артикул `…/c/SS27/a/demo-ss27-01`:** «Конкретный демо-slug из наших e2e; часть состояния может быть в localStorage — это сознательно для стабильного демо, не маскируем».
- **Tech pack (если есть):** «Отдельный пилот: вложения, индекс, handoff — работает только при собранном env; иначе пропускаем и не продаём как готовое».
- `**/brand/collections`:** «Витрина сезона перед B2B; один узкий showcase выбираем заранее из меню, не ведём инвестора по всем JOOR/NuORDER-экранам».
- `**/brand/b2b-orders` + карточка:** «Операционные заказы — единый read-model и API; это не отдельный Excel-трекер».
- `**/brand/integrations` (коротко):** «Карта подключений; каталог-summary для нарратива — с явным demo-источником».
- `**/brand/messages`, `/brand/calendar`:** «Сквозной слой переговоров и сроков; не смешиваем все типы календарей в одну легенду; realtime enterprise на все роли — отдельная история».
- **Shop (если заходим):** «Тот же операционный заказ с стороны ритейла — зеркало контракта».
- **Закрытие:** «Три столба уже завязаны на смоки и preflight в репозитории; следующий шаг — сузить меню под investor mode, не переписывая домен».
