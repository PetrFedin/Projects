# Тикеты: Core3 — коммуникации в контексте B2B-заказа

Кросс-функциональные пункты (FE / API / UX), не дублируют целиком goal фазы 999.1 в ROADMAP — лежат здесь как **готовые формулировки для issue** и **очередь внедрения**.

## Очередь

1. **FE + хелпер** — автономно, быстрый выигрыш и единообразие ссылок.
2. **API** — после появления модели (blocked до контракта/данных).
3. **UX** — в паре с (2) или сразу после согласования прототипа API.

---

## Core3-FE

**Заголовок:** `[Core3][FE] B2bOrderCommsContextButtons — опциональный контекст и единый сборщик href`

**Описание:** Опциональный overlay-контекст для кнопок/действий связи в B2B-заказе: коллекция, артикул / `stagesSku`, этап, PO ref, sku. Единый хелпер сборки `href` (и при необходимости query), без дублирования логики по экранам.

**Готово когда:** контекст только при известных данных; без данных — прежнее поведение; один модуль отвечает за формат URL; в JSDoc перечислены опциональные поля.

**Зависимости:** нет. Уместно отдельным небольшим PR.

---

## Core3-PROD (blocked)

**Заголовок:** `[Core3][API] Messages / Calendar — привязка к orderId и фильтры по контексту`

**Описание:** Привязка треда и задачи к `orderId`; опционально `collectionId`, `catalogStageId`, `poRef`. Списки с фильтрами по query; согласовать имена параметров и семантику «пусто = не фильтровать».

**Готово когда:** задокументированный контракт запросов/ответов; поведение при частичном контексте; согласованность с FE-хелпером из Core3-FE.

**Зависимости:** модель на бэкенде.

---

## Core3-UX

**Заголовок:** `[Core3][UX] Сообщения/календарь — пустой результат и сброс фильтров при смене контекста`

**Описание:** Правила «пустой результат» (копирайт, CTA). При смене контекста (заказ, артикул/этап/PO) — сброс или пересборка фильтров, чтобы не показывать чужую выдачу.

**Готово когда:** сценарии смены контекста / загрузки / нет совпадений; согласование с API из Core3-PROD.

**Зависимости:** логично после или вместе с прототипом API (2).
