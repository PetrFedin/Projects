## Backlog

### Phase 999.1: Боевой PO ↔ ядро №1 (матрица, stepId, бандл) (BACKLOG)

**Goal:** Связать заказы поставщику (PO) из ERP/внешнего API с единым контуром ядра №1: этапы каталога (`stepId`), выходы матрицы unified flow, черновики PO во вкладке «План · PO» у артикула — без ломки модели при подключении реального бэкенда.

**Requirements (черновик для promote):**

1. **Контракт API:** CRUD и статусы PO, стабильные идентификаторы, версионирование при расхождении с клиентом.
2. **Маппинг:** PO ref → `stepId` каталога (как минимум этап `po` и связанные узлы графа); привязка к строкам матрицы коллекции и к полям бандла артикула.
3. **Расхождения:** явная обработка черновика в бандле vs подтверждённого PO в ERP (diff UI, выбор источника правды по сценарию).
4. **Порядок внедрения:** сначала стабильный UX и запись в существующий слой (локальный unified flow / уже имеющийся API без расширения схемы), затем подключение боевого PO без перелома клиентской модели.
