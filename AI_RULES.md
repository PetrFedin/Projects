# AI_RULES.md — Правила ИИ

## Экономия usage (приоритет)

1. **Context Control**: Читать только нужные файлы и только диапазоны строк. Никогда не читать файлы >300 строк целиком.
2. **Минимум инструментов**: grep > read_file. Один точечный вызов лучше нескольких «на всякий случай».
3. **Короткие ответы**: Лаконично. Без дублирования кода в ответе.
4. **Редактирование**: search_replace с минимальным контекстом, не переписывать файлы целиком.
5. **Правила**: .cursor/rules/token-economy.mdc и no-monsters.mdc — соблюдать всегда.

## Общее

1. **Rule-based over LLM**: Сначала простая логика, кэш, поиск. LLM — последний рубеж.
2. **Token Guard**: Ограничение `max_tokens`. Избегание повторных вызовов.
3. **Caching**: Redis/LocalStorage для кэширования ответов ИИ.
