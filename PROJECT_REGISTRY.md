# Реестр проекта (Project Registry)

## 1. Общее описание проекта
Масштабная B2B/B2C платформа для Fashion-индустрии. Включает управление производством (создание Технических Заданий, Bill of Materials, Паспортов артикулов), виртуальный шоурум, B2B-портал для оптовых заказов, интеграции с ERP/PLM системами и внутренние инструменты для сотрудников бренда.

## 2. Текущий стек
**Frontend:**
- Фреймворк: Next.js 15.3.x (App Router), React 19
- Стилизация: Tailwind CSS, Radix UI (shadcn/ui), `lucide-react`
- Язык: TypeScript
- Тестирование: Playwright (E2E), Jest
- Дополнительно: Интеграция с AI (Google Gemini, Genkit)

**Backend:**
- Фреймворк: FastAPI (Python)
- Инфраструктура: Sentry, облачные сервисы

**База данных / Хранилище:**
- PostgreSQL
- Local JSON fallback (для dev-среды)

## 3. Структура папок
Основные директории `_ai-share/synth-1-full/src/`:
- `app/` — роутинг страниц и API. Разделен по доменам (`admin/`, `brand/`, `client/`, `shop/`, `factory/`, `api/`).
- `components/` — UI-компоненты. Разделены на доменные (`brand/production/`, `b2b/`, `client/`) и общие (`ui/`, `layout/`).
- `lib/` — доменная логика (DDD), абстракции БД (`server-store`), интеграции (`b2b/`), утилиты и типы.
- `hooks/` — переиспользуемые React hooks (состояния, данные, RBAC).

Директория `app/api/v1/endpoints/`:
- Содержит логику Python/FastAPI бэкенда (разбита по доменам: `orders.py`, `logistics.py`, `product.py` и др.).

## 4. Реализованные функции
- **Управление разработкой (Workshop 2.0 Dossier):** создание ТЗ, спецификации материалов (BOM), цифровые подписи (Signoffs), система утверждения этапов (Gates).
- **B2B портал:** оптовые заказы, управление каталогом, распределение стоков, интеграции с Joor, NuOrder, Sparklayer.
- **Ролевая модель:** управление доступом на уровне UI и API (RBAC).

## 5. Частично реализованные функции
- **Раздел "Техническое задание" (Tech Pack):** Мощная бизнес-логика, но UI находится в стадии рефакторинга и стабилизации.
- **Авторизация:** Присутствуют mock-провайдеры, смешанные с реальной логикой.
- **B2B интеграции:** Часть переведена в `archive/`, часть использует mock-данные.

## 6. Нереализованные функции
- Полноценная production-интеграция AI-чата (Syntha Chat).
- Полный отказ от mock-данных в Virtual Showroom / 3D-модулях.

## 7. Связи между модулями
- `lib/production/` (Core) <-> `components/brand/production/` (UI): Строгая зависимость UI от доменных сервисов и типов (например, `Workshop2DossierPhase1`).
- B2B API <-> Интеграционные клиенты (`fashion-cloud-client.ts` и др.).

## 8. Основные пользовательские сценарии
- **Создание коллекции и артикула:** Выбор категории (L1-L3), заполнение паспорта, добавление эскизов.
- **Разработка ТЗ (Dossier):** Заполнение состава материалов, конструктивных особенностей, назначение исполнителей, финальная подпись и передача на фабрику.
- **B2B заказ:** Просмотр каталога, добавление в корзину, оформление оптового заказа с учетом скидок и правил.

## 9. Статус frontend
🟡 **Частично работает (Активная разработка):** Большое количество untracked-файлов. Ведется рефакторинг UI-компонентов ТЗ. Есть предупреждения React (например, дублирование ключей в навигации).

## 10. Статус backend
✅ **Работает:** FastAPI бэкенд хорошо структурирован. Next.js Route Handlers также функционируют, но требуют чистки от устаревших B2B-роутов.

## 11. Статус базы данных
🟡 **Частично работает:** Используется PostgreSQL, но есть потенциально опасный фоллбэк на локальный JSON-файл (`fs.writeFileSync`), который недопустим в production.

## 12. Статус авторизации
🟡 **Частично работает:** Используются mock-данные и заглушки для ролей. Необходима миграция на production-провайдер.

## 13. Статус API
🟡 **Частично работает:** Много заглушек. Перегруженность монорепозитория Next.js роутами. 

## 14. Известные баги
- `Warning: Encountered two children with the same key, /catalog.` в `header.tsx`.
- Неконсистентность отступов и стилей в "Паспорте артикула" после внедрения универсального инпута (исправлено в последнем коммите/откате).

## 15. Технический долг
- Гигантские UI-компоненты (по 500-700 строк), которые сложно поддерживать.
- Мок-данные (файлы `*-demo-data.ts`, `repositories/mock/`) просочились в рабочий контур.
- Использование локального файлового хранилища как замены БД в Next.js.

## 16. Что нельзя менять без согласования
- **Дизайн и Layout:** Любые попытки стандартизировать верстку (универсальные инпуты) должны учитывать локальный контекст разделов ТЗ.
- **Бизнес-логика в `lib/production/`:** Это ядро системы, любое изменение влияет на процесс утверждения артикулов.

## 17. Последние изменения
- Оптимизирована структура раздела ТЗ: закреплен порядок вкладок (Паспорт → Материалы → Конструкция → Задание).
- Сущность "Визуал" скрыта из навигации и интегрирована в "Конструкцию" (включая логику гейтов и подписей).
- Сокращены длинные описания к полям в Паспорте (добавлен `line-clamp-1`).
- Убраны лишние групповые кнопки ("Снять все", "Все позже") из пользовательского ТЗ для упрощения интерфейса.
- Вкладка "Задание" очищена от каталожных полей (Sample Intake) и теперь служит только финальным экраном передачи.
- Оформление (стилистика карточек) "Паспорта артикула" выровнено по стилю с разделом "Материалы и состав", как просил пользователь.
