# Архивы и legacy в монорепо `Projects`

Цель: не смешивать **канон** (`_ai-share/synth-1-full`, Python `app/`) с локальными копиями и обломками старого фронта.

## Что является каноном (не архивировать как «мусор»)

| Путь | Назначение |
|------|------------|
| `_ai-share/synth-1-full/` | Единственный Next.js Fashion OS, CI, E2E |
| `app/`, `tests/`, `pyproject.toml`, `scripts/` (кроме архивных), `static/` | Backend FastAPI и инфраструктура монорепо |
| `pw-demo/`, `tools/superpowers/` | Субмодули (см. `docs/SUBMODULES.md`) |
| `docs/` в корне | Политика монорепо, миграции, тулкиты |

## Что обычно **не нужно** в повседневной работе (архив / вне репо)

| Путь | Почему | Действие |
|------|--------|----------|
| `synth-1/` (если появится снова) | Субмодуль удалён; канон — `full` | Архив → удалить/оставить только вне репо после бэкапа. Скрипт: `bash scripts/archive-local-legacy-frontend.sh` |
| `.worktrees/*` | Локальные зеркала веток (часто гигабайты), не часть канона | Не коммитить. В корневом `.gitignore` игнорируется. При необходимости — `tar` во внешний архив и `rm -rf .worktrees` |
| Корневой `src/` (TS: `lib/cn.ts`, `design/tokens.json`, …) | Нет корневого `package.json`, нет App Router — **не** текущий фронт | Перенос уникального в `_ai-share/synth-1-full` **diff’ом**, затем архив + удаление из рабочей копии (осторожно: имя `src/` совпадает с другими проектами — ориентир: наличие `src/lib/cn.ts` / `src/design/tokens.json`) |
| `.ai_context/`, `.ai_reports/` | Уже в `.gitignore`, артефакты агентов | Периодически чистить локально |
| `synth1.db`, `dev_server.log`, `logs/`, `.venv/`, `.pytest_cache/` | Локальная среда / кэш | Не коммитить; при желании удалить локально |
| `restore_showroom.py`, `update_page_for_b2c.py` (пустые в корне) | Заглушки без содержания | Удалить или перенести в `scripts/` с реализацией — по решению владельца |

## Что **переносить в full** перед архивом (если ещё не перенесено)

Только осознанный **diff**, не `cp` всего дерева:

- Уникальные токены/темы из корневого `src/design/tokens.json` → в дизайн-систему `full`.
- Полезные хуки/утилиты из корневого `src/lib/*` → в `full/src/lib/` с выравниванием импортов и линтера.
- Любой контент из старого `synth-1/`, которого нет в `full`, — по чеклисту `docs/MIGRATION_FULL_CUTOVER.md` и `full/docs/CANONICAL_FULL.md`.

Дублирующие **корневые** `.md` (планы, карты коллекций) имеет смысл со временем **свести к одному источнику** в `full/docs/` или `Projects/docs/`, чтобы не править дважды.

## Команды

```bash
# Только synth-1/ (если каталог есть)
ARCHIVE_PATHS="synth-1" bash scripts/archive-local-legacy-frontend.sh

# Корневой legacy src/ — только после переноса нужного в full (флаг обязателен)
ARCHIVE_INCLUDE_LEGACY_SRC=1 bash scripts/archive-local-legacy-frontend.sh

# Другой каталог назначения
ARCHIVE_DEST="$HOME/Backups" ARCHIVE_PATHS="synth-1" bash scripts/archive-local-legacy-frontend.sh
```

После успешного `tar` и проверки списка файлов внутри архива можно удалить исходные каталоги **вручную**.

## После запуска у себя

Архивы по умолчанию — **`~/Archive/Projects-legacy/`** (не коммитятся). Имена файлов с датой-штампом смотрите в выводе скрипта или `ls` в этой папке.

Если вы удалили **`.worktrees/*`** после `tar`, при необходимости ветку можно снова получить через **`git worktree add`** из основного клона.
